# Vaked lowering (Goal 3) — design

- **Date:** 2026-06-10
- **Status:** Approved (brainstorm) → implementing via subagent-driven execution
- **Goal 3** of the language session: the lowering story — how the validated typed semantic graph (Goal 2) compiles to artifacts. (Goal 1 = grammar v0.2 ✅; Goal 2 = type system ✅.)

## Decisions

1. **Architecture — hybrid: direct artifacts + a Nix spine.** Vaked directly emits the boring, inspectable config/policy artifacts it owns AND a `flake.nix` + NixOS module spine that wires/builds/deploys them. ("Complement above Nix"; "compile to boring artifacts".)
2. **Scope — framework + Nix spine + 3 exemplars.** The lowering *framework* (typed graph → emitters → artifacts + provenance + determinism) and the always-present Nix spine are specced concretely, plus three exemplar direct-emitters that prove it end-to-end: **generated `RUNTIME.md` docs**, **a Zig daemon config**, **CrabCC index + SQLite/JSONL catalog**. eBPF policy, OTel config, systemd units, and surface launchers get a defined emitter *interface* with deferred mappings (their consuming daemons aren't designed yet).
3. **Provenance — headers + decl-level manifest.** Every artifact carries a `# generated by Vaked from <file>:<decl> — do not edit` header, AND lowering emits a machine-readable `.vaked/provenance.json` mapping each artifact (+ key regions) → {source file, decl, span, emitter, inputs-hash}.

**Ratified defaults:** flake outputs = `nixosModules.<runtime>`, `packages` (built Zig daemons), `devShells`, `apps` (surface launchers), inputs pinned; direct artifacts land in `gen/`, committed + inspectable; the 3 exemplars as listed.

## Model

- **Pipeline placement.** Lowering is the final stage after Goal 2's check: validated typed semantic graph → artifacts. It runs ONLY on a graph that type-checked ("validate before generating").
- **Purity/totality/hermeticity.** Lowering is a pure, total function of (validated graph + pinned inputs): same graph ⇒ byte-identical artifacts; no network/IO during lowering (fetching is the Nix build's job, pinned via `flake.lock` from `trust = pinned{…}`).
- **Emitters.** One emitter per target; each is a pure function (graph, relevant nodes) → files + provenance entries. A registry maps target → emitter; adding an emitter touches no core. The Nix-spine emitter always runs; direct emitters run per declared `emit` targets (`index.emit = [catalog.jsonl, nix.derivation]` drives selection — already expressible, **no grammar change**).
- **Nix spine.** `flake.nix` (outputs above) + NixOS module(s) that wire the OTP/Zig daemons and reference the direct-emitted config artifacts as installed files.
- **Escape hatch.** `nix("…")` app passes through verbatim to the Nix emitter, with provenance.

## Review criteria

T1 internally consistent with 0011 + the manifesto (pure/total/hermetic argued; no eval-time IO anywhere); emitter interface genuinely closed (an emitter can't smuggle computation/IO); the 3 exemplar mappings concrete enough to implement; T2 fixtures conform to T1's mappings + headers + provenance schema (spot-derivable from operator-field.vaked's decls); no scope creep into daemon implementation or the deferred targets; no grammar changes.
