## 1. Objective

Implement the **`hcp.control` schema wire codec** — `encode`/`decode` for the seven
control-plane frames defined in RFC 0005 §1, plus the `ControlPlane` service binding —
in the `litany` Rust crate, with a golden-vector + canonical-rejection test suite that
passes under `cargo test` on M1.

Concretely, S4 delivers the byte-level representation of:

| Frame | Class | RFC 0005 §1 |
|-------|-------|-------------|
| `PauseControl` | request | lines 71–76 |
| `ResumeControl` | request | lines 79–83 |
| `SetIntervalControl` | request | lines 86–91 |
| `StepControl` | request | lines 94–98 |
| `RewindControl` | request | lines 104–109 |
| `ConsumerCheckpointEvicted` | request | lines 114–121 |
| `ControlAck` | response | lines 124–129 |

plus the two enums `ControlScope` and `ControlRefusal` (RFC 0005 §1 lines 53–68) and the
`ControlPlane` service with its six `call` bindings (lines 131–138).

### 1.1 Scope boundary (the spine of this sprint)

WP3-S4 is the **wire/codec representation only**. The semantics around these frames are a
**different work package** (`agent-supervisord`, per the control-plane design doc plan
items 3–5, [`docs/superpowers/specs/2026-06-12-control-plane-design.md`](2026-06-12-control-plane-design.md)).
The codec is what *produces and parses bytes*; the supervisor is the downstream *consumer*
that enforces meaning.

**In scope (S4):**
- Rust structs/enums mirroring the RFC 0005 §1 schema.
- `encode(&Frame) -> Vec<u8>` and `decode(kind, &[u8]) -> Result<Frame, DecodeError>` for
  each frame **body** (the `>= @1` tagged fields), applying RFC 0002 §6.2 ascending-tag
  emission and §6.3 canonical default-omission.
- The `ControlScope` / `ControlRefusal` enum codec (bare varint, §6.7.1) including
  unknown-case preservation (forward-compat, §6.7.1 / §8).
- The `ControlPlane` service binding as a Rust type that maps `(method, request-type) ->
  response-type` (RFC 0002 §4.3), so the routing layer can dispatch.
- Hand-derived golden byte vectors (§4) and canonical-rejection tests (§4.3).

**Out of scope (NOT S4 — do not implement here):**
- Graceful-pause / step / interval / rewind **semantics** (RFC 0005 §2, §3) — supervisor.
- `preceptord` authority checks (RFC 0005 §2.3, Security considerations) — supervisor + policy.
- `topology_epoch` **validation** / `stale_epoch` enforcement (RFC 0005 §2.4, §7) — supervisor.
  S4 only encodes/decodes the `topology_epoch` *field*; it never decides currency.
- Writing `control_action` to `eventd` (RFC 0005 §8) — supervisor (WP3-S5 integrates eventd).
- Idempotency / nonce de-dup (RFC 0005 §2.5, §4) — supervisor.
- The **frame header** (`kind`/`corr`/`stream`/`seq`/`end`) — owned by the wire layer
  (RFC 0003 §4.4.1), delivered in WP3-S2. S4 bodies start at `@1` (RFC 0005 §1 line 45:
  "Header fields … are implicit; tags begin at `@1`").
- The frame-record length prefix and `max_frame` guard (RFC 0003 §4.1–§4.3) — WP3-S2.

## 2. Inputs / oracles

### 2.1 Normative inputs
- **RFC 0005 §1** — the `hcp.control` schema (frame field names, types, tags, enums,
  service). This is the authoritative shape.
- **RFC 0002 §6.2** (records/frames: field ordering & tags) — fields emitted in *strictly
  ascending tag order*; decoders reject out-of-order/duplicate tags; tag `@0` reserved.
- **RFC 0002 §6.3** (defaults, optionals, canonical omission) — the rule that makes the
  encoding canonical; see §3.3 below for how it applies to every control field.
- **RFC 0002 §6.1** (varints), **§6.4** (bool / string-NFC), **§6.5** (hash), **§6.7**
  (enums encode as a bare varint; unknown cases preserved) — the primitive layer.
- **RFC 0002 §5.1** — `uuid` = 16 raw bytes, *no* length prefix, *no* endianness applied.
- **RFC 0003 §4.4.1** — frame header field layout (consumed by S2; S4 must agree that
  `ControlAck` is `kind=response (1)` and the six requests are `kind=request (0)`, matching
  RFC 0005 §1's `request`/`response` class annotations and RFC 0002 §4.3's `call` binding).

### 2.2 The oracle gap (critical — read before writing tests)
RFC 0005 ships **no byte vectors**. The oracle for S4 is *derivational*: apply RFC 0002
§6.2/§6.3 to each RFC 0005 frame shape, **hand-derive the canonical bytes, and pin them as
the golden set** committed in the test file. The existing primitive vectors that anchor the
derivation are:
- Unsigned varint vectors: RFC 0002 §6.1.1 (and already implemented + tested in
  `protocol/hcp/hcpbin/src/lib.rs`, `tests::uvarint_golden`).
- Enum vectors: RFC 0002 §6.7.1 (lines 808–823) — `info=0 -> 00`, `warn=1 -> 01`,
  unknown `5 -> 05`.
- Default-omission vectors: RFC 0002 §6.3.1 (lines 622–669) — the worked `Outer`/`Config`
  examples are the template for §3.3 below.
- String vectors: RFC 0002 §6.6.1 (`"a" -> 01 61`, `"bc" -> 02 62 63`).

A fully-worked seed vector for `PauseControl` is given in §4.1 so the test author has a
reference derivation. **Each derived vector MUST be re-checked against §6.3** by the
implementer; the default-omission cases (§3.3) are where mistakes happen.

### 2.3 Code oracle (already shipped)
`protocol/hcp/hcpbin/src/lib.rs` (WP3-S1, done) provides the primitive `Reader`/`Writer`:
`put_uvarint`/`get_uvarint`, `put_bool`/`get_bool`, `put_bytes`/`get_bytes`,
`put_hash`/`get_hash` (`Hash{algo,digest}`), `put_raw`/`get_raw`, and whole-value
`decode_all` (rejects `TrailingBytes`, §6.8). The `DecodeError` enum (`UnexpectedEof`,
`Overlong`, `NonMinimal`, `WidthOverflow`, `InvalidBool`, `HashLengthMismatch`,
`LengthOverflow`, `TrailingBytes`) is the shared error surface; S4 extends it (§3.4) with
record-level variants. S4 reuses this crate; it does **not** re-implement varints.

## 3. Design

### 3.1 Field-type inventory (bounds the dependency set)

`hcp.control` is a **flat** schema: scalars + enums + optionals only. **No unions, no
lists, no maps, no nested records** (`ConsumerCheckpointEvicted.request_nonce` is `bytes`,
not a record). The complete set of field types across all seven frames:

| RFC 0002 type | Frames using it | Codec source |
|----------------|-----------------|--------------|
| `enum` (varint) | `ControlScope` (Pause/Resume/SetInterval/Step), `ControlRefusal` (ControlAck) | S1 `get/put_uvarint` + §6.7 unknown-case handling (S4 thin wrapper) |
| `string` | `target`, `reason` (CCE) | **S2** `string` codec (NFC, Unicode 15.1.0, §6.4) |
| `string?` | `PauseControl.reason`, `ControlAck.detail` | S2 string + S4 optional handling |
| `u32` | `SetIntervalControl.interval_ms` | S1 uvarint, width-bounded (`get_u32`) |
| `u64` | every `topology_epoch`, `rewind_to_step`, `min_required_step` | S1 `get_u64`/`put_uvarint` |
| `u64?` | `ControlAck.logged_seq` | S1 uvarint + S4 optional handling |
| `bool` | `ControlAck.applied` | S1 `get/put_bool` |
| `bytes` | `ConsumerCheckpointEvicted.request_nonce` | S1 `get/put_bytes` |
| `hash` | `RewindControl.rewind_to_hash` | S1 `get/put_hash` |
| `uuid` | `RewindControl.producer`, `CCE.producer_id`/`consumer_id` | **§5.1 — 16 raw bytes; NOT in hcpbin yet** (see §3.5) |

Note: §6.7 *union* codec is **not** needed here — do not pull it into S4.

### 3.2 The record codec (the real per-field machinery)

Every frame body is a §6.2 record: present fields emitted in **strictly ascending tag
order**, each as `varint(tag) ++ value-bytes`. S4 needs a small, frame-specific
encode/decode per frame. Recommended shape:

```rust
// protocol/hcp/litany/src/control.rs

pub struct PauseControl {
    pub scope: ControlScope,        // @1  enum, default unspecified(0) -> omit
    pub target: String,             // @2  string, default "" -> omit
    pub topology_epoch: u64,        // @3  u64,    default 0  -> omit
    pub reason: Option<String>,     // @4  string?, absent -> omit
}

impl PauseControl {
    pub fn encode(&self, w: &mut Writer) {
        // ascending tag order; §6.3 default-omission per field:
        if self.scope != ControlScope::Unspecified { w.put_uvarint(1); put_enum(w, self.scope); }
        if !self.target.is_empty()                 { w.put_uvarint(2); put_string(w, &self.target); }
        if self.topology_epoch != 0                { w.put_uvarint(3); w.put_uvarint(self.topology_epoch); }
        if let Some(r) = &self.reason              { w.put_uvarint(4); put_string(w, r); }
    }
    pub fn decode(r: &mut Reader) -> Result<Self, ControlDecodeError> {
        // tag-loop: read varint(tag), dispatch, enforce ascending+no-dup,
        // substitute defaults for omitted non-optional fields (§6.3 line 619).
    }
}
```

The decode tag-loop is shared structure across all seven frames; factor a helper
(`read_next_tag` returning `Option<u64>` with ascending/duplicate enforcement) rather than
a generic derive macro (keep it simple — seven hand-written frames, per CLAUDE.md "no
abstractions for single-use code"; a `derive` is over-engineering for seven flat structs).

### 3.3 Canonical default-omission, per field (the load-bearing detail)

Apply RFC 0002 §6.3 (lines 571–620). For **non-optional** fields, a value equal to the
implicit default **MUST be omitted** (line 578); decoders substitute the default
(line 619). For **optional** (`T?`) fields, absent -> omit; present (even if value equals a
default) -> emit (line 577). Field-by-field:

| Field | Type | Default behavior |
|-------|------|------------------|
| `scope` | `ControlScope` enum | implicit default = case 0 `unspecified` (§6.3 line 593). `scope=unspecified` -> **@1 omitted**. This is exactly the RFC 0005 §1 line 51 / Security "default-deny by encoding": an omitted scope decodes to `unspecified`, which the supervisor refuses. `scope=agent(2)` emits `@1`. |
| `target` | `string` | default = empty (§6.3 line 591). `target=""` -> **@2 omitted** (decodes to `""`, which the supervisor refuses `unknown_target`). |
| `topology_epoch` | `u64` | default = 0 (line 589). `topology_epoch=0` -> **@3 omitted**. (Epoch 0 is unusual but the codec must omit it canonically.) |
| `interval_ms` | `u32` | default = 0 -> **@4 omitted** when 0. (RFC 0005 §1 line 90: `interval_ms=0` is the omitted default, refused `invalid_interval` by the supervisor.) |
| `reason` (Pause) | `string?` | absent -> omit; present (even `""`) -> emit `@4` (line 577). |
| `applied` (Ack) | `bool` | default = false (line 588). **`applied=false` -> @1 omitted entirely**; body of a refusal is just `@2 reason`. Counterintuitive — pin a golden vector. |
| `reason` (Ack) | `ControlRefusal` enum, `= unspecified` declared default | default omitted -> **@2 omitted** when `unspecified`. A successful `applied=true` Ack omits both `@1` (false… no, applied=true is non-default, emits) — see §4.2 for the exact bytes. |
| `logged_seq` (Ack) | `u64?` | absent -> omit; present -> emit `@3` even if 0 (line 577). |
| `detail` (Ack) | `string?` | absent -> omit; present -> emit `@4`. |
| `producer` etc. | `uuid` | **no implicit default** (§6.3 line 596): a non-optional `uuid` is **always emitted** (line 601). All `uuid` fields in `RewindControl`/`CCE` always appear on the wire. |
| `rewind_to_hash` | `hash` | **no implicit default** (line 596): always emitted. |
| `rewind_to_step`, `min_required_step` | `u64` | default 0 -> omitted when 0 (rare for a real rewind, but canonical). |
| `request_nonce` | `bytes` | default = empty -> omitted when empty (but a real nonce is never empty; supervisor concern). |
| `reason` (CCE) | `string` (non-optional, RFC 0005 §1 line 119) | default empty -> omitted when `""`. |

Decoders MUST treat "tag absent" and "tag present == default" as the same value for
non-optional fields (§6.3 line 619) — but since encoders MUST omit, a *present default* on
a non-optional field is a **non-canonical encoding** and SHOULD be rejected for strictness
(see §4.3 rejection tests; this mirrors hcpbin's `NonMinimal` posture).

### 3.4 Errors

Reuse `hcpbin::DecodeError` for primitive failures. Add a thin record-level error in
`control.rs`:

```rust
pub enum ControlDecodeError {
    Primitive(hcpbin::DecodeError),  // varint/bool/bytes/hash/string failures
    TagOutOfOrder { prev: u64, got: u64 },     // §6.2 rule 1
    DuplicateTag(u64),                          // §6.2 rule 1
    UnknownTagInBody(u64),    // a tag with no field in this frame (see note)
    NonCanonicalDefault(u64), // non-optional field present with default value (§6.3)
    ReservedTagZero,          // @0 appeared in a body (§6.2 rule 3)
    UnknownEnumCase { field_tag: u64, case: u64 }, // see §3.6 — preserved, not error, for fwd-compat
}
```

Note on `UnknownTagInBody`: RFC 0002 §6.2 line 557 says self-delimitation lets a decoder
*skip and preserve* unknown nested shapes for round-trip — but the control frames are flat
and every field is a fixed primitive, so an unknown trailing tag in a *frozen* control
frame is ambiguous to skip (a bare varint field has no length prefix). **Decision (stated
assumption):** for S4, treat an unknown tag in a control-frame body as a decode error
(`UnknownTagInBody`), because RFC 0005 is the frozen control schema and the wire pins the
schema digest (RFC 0003 §5.4) — a peer sending an unknown control tag is using a different
schema, which is a frame-level error, not a forward-compat case. (Enum *case* evolution is
handled separately, §3.6.) Revisit if RFC 0005 adds the open-question `steps: u32` field
(§ Open questions line 547) — that would be a schema-version event handled by digest
negotiation, not in-band tolerance.

### 3.5 `uuid` codec — a prerequisite gap

`uuid` (RFC 0002 §5.1 line 330) = **16 raw bytes, no length prefix, no endianness**. It is
**not in `hcpbin` today** — `lib.rs` lines 488–498 list TODOs (string, f32/f64, records,
unions) but **not uuid**. It is trivial over `put_raw(&[u8;16])` / `get_raw(16)`, but it
must exist somewhere before S4 can encode `RewindControl`/`ConsumerCheckpointEvicted`.

**Stated assumption / dependency:** S4 will add `put_uuid`/`get_uuid` to `hcpbin` (16-byte
fixed) if S2 has not. This is the cleanest home (it is a §6 primitive). Flag in §7. The
decode MUST require exactly 16 bytes available (`get_raw(16)` already errors
`UnexpectedEof` otherwise).

### 3.6 Enum codec (`ControlScope`, `ControlRefusal`)

Per RFC 0002 §6.7.1 (lines 808–823): an enum encodes as a **bare varint** of the case
value. Known cases map to the Rust enum; an unknown case (from a future RFC 0005 revision)
is **preserved** for byte-for-byte round-trip (line 822: `unknown(5) -> 05`). Model as:

```rust
pub enum ControlScope { Unspecified, Runtime, Agent, Unknown(u64) }
```

`put_enum` emits the underlying varint; `get_enum` maps `0/1/2` to the named cases and any
other value to `Unknown(n)`, re-emitting `n` verbatim. (This matters for `ControlRefusal`
too — a future refusal reason must round-trip through an older decoder.) Round-trip of an
`Unknown` case is a test (§4.3).

### 3.7 Service binding

RFC 0005 §1 lines 131–138 define `service ControlPlane` with six `call` bindings. Per
RFC 0002 §4.3 (line 268), `call (Req) -> Resp` binds a `request` frame to a terminal
`response`; **all six** bind to `ControlAck`. S4 expresses this as a Rust enum the routing
layer (S3) dispatches on:

```rust
pub enum ControlRequest {
    Pause(PauseControl), Resume(ResumeControl), Slow(SetIntervalControl),
    Step(StepControl), Rewind(RewindControl), EvictCheckpoint(ConsumerCheckpointEvicted),
}
// all -> ControlAck
```

S4 does **not** implement dispatch/routing (that is S3's layer); it provides the typed
request enum and `ControlAck` so S3 can wire them. The mapping table (method name ->
request variant -> `ControlAck`) is the deliverable, matching RFC 0005 §1 lines 132–137.

## 4. Golden vectors (derived — S4 deliverable)

These are derived from RFC 0002 §6.2/§6.3 and pinned in `tests/control_golden.rs`. The
implementer MUST re-verify each by the §6.3 rules. Bytes are hex; the body starts at `@1`
(no frame header — that is the S2 wire layer, RFC 0003 §4.4.1).

### 4.1 `PauseControl` (fully-worked seed)

`PauseControl{ scope: agent, target: "worker-1", topology_epoch: 5, reason: "GC drain" }`
(the RFC 0005 §5 worked example, lines 287–292), ascending tag, §6.3 omission:

- `@1 scope=agent(2)` (non-default) -> `01` (tag) `02` (varint case 2)
- `@2 target="worker-1"` (len 8) -> `02` (tag) `08` (len) `77 6f 72 6b 65 72 2d 31`
- `@3 topology_epoch=5` (non-default) -> `03` (tag) `05` (varint)
- `@4 reason="GC drain"` (present, len 8) -> `04` (tag) `08` (len) `47 43 20 64 72 61 69 6e`

**Encoded body:** `01 02 02 08 77 6f 72 6b 65 72 2d 31 03 05 04 08 47 43 20 64 72 61 69 6e`

Default-omission variant — `PauseControl{ scope: unspecified, target: "", topology_epoch: 0,
reason: absent }` (every field at default) -> **empty body** (zero bytes). This is the
canonical "default-deny by omission" frame (RFC 0005 Security line 501–503).

### 4.2 `ControlAck` (the counterintuitive cases — pin these)

- `ControlAck{ applied: false, reason: stale_epoch(3) }` (a refusal):
  `applied=false` is the bool default -> **@1 omitted**; `reason=3` non-default ->
  `02 03`. **Encoded body:** `02 03` (tag @2, enum 3). *(Same shape with `reason` swapped:
  `denied=1 -> 02 01`, `unknown_target=2 -> 02 02`, `not_paused=4 -> 02 04`,
  `invalid_interval=5 -> 02 05`, `invalid_rewind=6 -> 02 06`.)*
- `ControlAck{ applied: true, reason: unspecified, logged_seq: 12345 }` (an applied ack):
  `applied=true` non-default -> `01 01`; `reason=unspecified(0)` declared default ->
  **@2 omitted**; `logged_seq?=12345` present -> `03` + varint(12345). 12345 = 0x3039 ->
  LEB128 minimal `b9 60`. **Encoded body:** `01 01 03 b9 60`.
- `ControlAck{ applied: true, logged_seq: 0 }` (present optional == 0): `logged_seq?` is
  optional and present, so emitted even though 0 (§6.3 line 577): `01 01 03 00`.
- `ControlAck{ applied: false, reason: invalid_rewind(6), detail: "hash mismatch…" }`:
  `02 06` then `04` + string(detail). (RFC 0005 §5 line 409 example.)

### 4.3 `SetIntervalControl`

`SetIntervalControl{ scope: agent, target: "worker-2", topology_epoch: 5,
interval_ms: 5000 }` (RFC 0005 §5 lines 356–361):
- `@1 -> 01 02`, `@2 "worker-2" -> 02 08 77 6f 72 6b 65 72 2d 32`, `@3 -> 03 05`,
- `@4 interval_ms=5000` -> `04` + varint(5000). 5000 = 0x1388 -> LEB128 `88 27`.
- **Encoded body:** `01 02 02 08 77 6f 72 6b 65 72 2d 32 03 05 04 88 27`
- Refused variant `interval_ms=0`: `@4` omitted (0 is the default), body
  `01 02 02 08 77 6f 72 6b 65 72 2d 32 03 05` — the supervisor (not S4) then refuses
  `invalid_interval`.

### 4.4 `RewindControl` (uuid + hash — both always emitted)

`RewindControl{ producer: <uuid>, rewind_to_step: 50, rewind_to_hash: SHA-256(<32B>),
topology_epoch: 5 }`:
- `@1 producer` (uuid, no default, always emitted) -> `01` + 16 raw bytes.
- `@2 rewind_to_step=50` -> `02 32`.
- `@3 rewind_to_hash` (hash, no default, always emitted) -> `03` + `01` (algo SHA-256) +
  `20` (len 32) + 32 digest bytes (per RFC 0002 §6.5 / hcpbin `put_hash`).
- `@4 topology_epoch=5` -> `04 05`.
- **Encoded body:** `01 <16B> 02 32 03 01 20 <32B> 04 05`.

### 4.5 `ResumeControl` / `StepControl`

Identical body shape (scope @1, target @2, topology_epoch @3). E.g.
`StepControl{ scope: agent, target: "worker-1", topology_epoch: 5 }` ->
`01 02 02 08 77 6f 72 6b 65 72 2d 31 03 05`.

### 4.6 `ConsumerCheckpointEvicted`

`{ producer_id, consumer_id, min_required_step, topology_epoch, reason, request_nonce }`
with two uuids (always emitted, @1/@2), `min_required_step` u64 @3, `topology_epoch` u64 @4,
`reason` string @5, `request_nonce` bytes @6. Body:
`01 <16B> 02 <16B> 03 <varint> 04 <varint> 05 <len> <utf8> 06 <len> <nonce-bytes>`.

## 5. Test plan

All functional tests run via `cargo test -p litany` (or `cargo test` in
`protocol/hcp/litany/`). **The control codec is pure Rust; 100% of S4's functional
verification is M1-local.** There is no namespace-syscall or Linux-only surface in S4.

### 5.1 M1-local (cargo test) — the entire S4 suite

1. **Golden encode.** For each vector in §4: build the struct, `encode`, assert bytes equal
   the pinned hex. (Catches tag-order and §6.3 omission bugs.)
2. **Golden decode.** For each vector: `decode` the pinned bytes, assert the struct equals
   the source. (Round-trip closure: `decode(encode(x)) == x` and `encode(decode(b)) == b`.)
3. **Default-omission correctness** (§3.3): the all-default `PauseControl` -> empty body;
   `ControlAck{applied:false,…}` omits @1; `ControlAck{applied:true}` with
   `reason:unspecified` omits @2; `interval_ms:0` omits @4. One assertion per row of §3.3.
4. **Optional vs default distinction**: `reason: Some("")` emits `@4 04 00`; `reason: None`
   omits @4 (mirrors RFC 0002 §6.3.1 `optional_attrs` example, lines 661–664).
5. **uuid / hash always-emitted** (§6.3 line 601): a `RewindControl` always has @1 and @3
   on the wire regardless of value (even an all-zero uuid is emitted, not omitted).
6. **Enum unknown-case round-trip** (§3.6): decode `02 07` as `ControlAck.reason` ->
   `Unknown(7)`; re-encode -> `02 07` byte-identical (RFC 0002 §6.7.1 line 822 posture).
7. **Canonical-rejection tests** (negative; these prove the decoder is strict):
   - Out-of-order tags (e.g. `@3` before `@2`) -> `TagOutOfOrder` (§6.2 rule 1).
   - Duplicate tag -> `DuplicateTag`.
   - `@0` in a body -> `ReservedTagZero` (§6.2 rule 3).
   - Non-minimal varint inside a field -> `DecodeError::NonMinimal` (reuses hcpbin, §6.1).
   - Non-canonical default present (e.g. `ControlAck` body `01 00 …` with `applied=false`
     emitted) -> `NonCanonicalDefault` (§3.3 strictness).
   - Truncated frame (declared string len > remaining) -> `DecodeError::UnexpectedEof`.
   - Trailing bytes after a whole-frame decode -> `DecodeError::TrailingBytes` (§6.8).
   - Unknown body tag in a flat control frame -> `UnknownTagInBody` (§3.4 decision).
8. **`u32`/`u64` width bounds**: `interval_ms` decoded from a varint > `u32::MAX` ->
   `DecodeError::WidthOverflow` (reuses `get_u32`, hcpbin `unsigned_width_overflow`).

### 5.2 What needs dev-cx53 / Linux — essentially nothing in S4

- **None of S4's correctness tests need dev-cx53.** The codec has no syscalls, no
  namespaces, no platform-specific behavior.
- The **≤10µs/frame perf number** (kickoff line 54) is a *dev-cx53, 8-core, no-alloc*
  measurement and belongs to **WP3-S6** (perf baseline), not S4. Do not invent a Linux-only
  test for S4. If a smoke `criterion` micro-bench is added in S4, it is informational only
  and not a gate; the binding gate is on dev-cx53 in S6.
- **Zig:** no Zig surface in S4. The `vakedz` Zig codegen (RFC 0002 §7.1, Appendix A.1)
  could in principle emit a `hcp.control` decoder, but there is **no cross-language interop
  vector in scope for S4**, so no `zig build -Dtarget=x86_64-linux` step is required here.
  (If a future sprint adds Zig↔Rust interop vectors, the §4 golden hex is the shared oracle;
  that is the natural extension point, not an S4 deliverable.)

## 8. Acceptance criteria

S4 is **done** when all of the following hold (all checkable on M1 via `cargo test`):

1. `protocol/hcp/litany/src/control.rs` defines all seven RFC 0005 §1 frames, both enums,
   the `ControlRequest` enum, and the `ControlPlane`→`ControlAck` mapping (§3.7), matching
   the RFC 0005 §1 schema exactly (field names, types, tags @1+).
2. `encode`/`decode` round-trip closes for every frame: `decode(encode(x)) == x` **and**
   `encode(decode(b)) == b` for every §4 golden vector (byte-for-byte; canonical per §6.8).
3. Every golden vector in §4 is pinned in `tests/control_golden.rs` and asserted
   byte-exact, **derived and re-verified against RFC 0002 §6.2/§6.3** (not from RFC 0005,
   which has none).
4. Canonical-omission is correct for all §3.3 rows — in particular the three counterintuitive
   cases (all-default body -> empty; `ControlAck{applied:false}` omits @1;
   `interval_ms:0` omits @4) each have a passing assertion.
5. The decoder is **strict**: every §5.1 rejection test passes (out-of-order/duplicate/`@0`
   tags, non-minimal varint, non-canonical default, truncation, trailing bytes, unknown
   body tag, width overflow) — each returns the specific typed error, not a panic.
6. Enum unknown-case round-trips byte-identically (§3.6 / RFC 0002 §6.7.1).
7. `uuid` and `hash` fields are always emitted (never omitted), verified by test (§5.1.5).
8. `cargo test -p litany` and `cargo test -p hcpbin` are green on M1; `cargo build -p litany`
   compiles on stable. No new `unsafe`.
9. The dependency assumptions in §7 (record-codec home: S2 vs S3; `uuid` home) are confirmed
   with the WP3 engineer and the resolution noted in the PR description (per CLAUDE.md §1 —
   surface, don't pick silently).
10. **Out of scope is respected:** S4 contains no pause/step/rewind semantics, no preceptord
    checks, no epoch validation, no eventd writes, no frame-header/length-prefix code
    (those are §1.1's other owners). A reviewer can trace every changed line to "encode or
    decode an `hcp.control` frame body."

### Non-goals / explicitly deferred
- Perf (≤10µs/frame) — WP3-S6 on dev-cx53.
- End-to-end control over a live Litany connection — WP3-S5+ and supervisor.
- The open RFC 0005 questions (`StepControl.steps` count line 547; uuid targets for the
  scheduling frames line 554) — schema-version events; not in the frozen S4 schema.
