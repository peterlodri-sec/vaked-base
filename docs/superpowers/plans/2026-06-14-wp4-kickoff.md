## sandboxd isolation backend — DECISION

**Chosen: native-exec** (Linux namespaces + cgroups v2 + seccomp; pure Zig syscall wrappers)

Rationale:
- Matches vakedos substrate (NixOS, kernel 6.x, cgroup-v2 mounted)
- No OCI runtime dep (no runc/containerd); no wasm/Firecracker overhead
- agent_guardd already uses eBPF cgroup hooks — sandboxd cgroups slot in alongside it
- Minimal: `CLONE_NEWNS | CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWUSER` + seccomp-bpf allowlist

Alternatives rejected: OCI (too heavy, adds runc dep), wasm (scope creep), Firecracker (VM overhead, overkill for S1).
