# vaked CTF simulation — v0

A deterministic, time-boxed capture-the-flag **simulation**: 2–4 teams race to solve scored
challenges within a 20 *simulated*-minute box. Pure stdlib, seeded, **replay-stable**, with a
game-theoretic analysis and a non-currency winner reward. Lives in `tools/ctf/`.

> Spec: `docs/superpowers/specs/2026-06-16-ctf-simulation-design.md`.

## Run it

```
python3 tools/ctf/ctf.py run [--teams N=4] [--seed S=1337] [--box-min 20]
                             [--strategies a,b,c,d] [--json] [--out timeline.jsonl]
python3 tools/ctf/ctf.py replay --events timeline.jsonl
python3 tools/ctf/test_ctf.py        # 17 tests, stdlib only
```

## Model

- **Arena** (`arena.py`) — challenges `{id, category, points, effort}` (effort = sim-minutes to
  solve at skill 1.0); a fixed 8-challenge default board; `time_box_min` (20), `first_blood_bonus`.
- **Teams** (`team.py`) — deterministic strategies (`greedy_points`, `greedy_easy`,
  `ratio_balanced`, `best_response`, `category_focus:<cat>`). Per-(team, category) **skill** ∈
  [0.5, 1.5) is `hashlib`-derived from `(seed, team, category)` — no RNG.
- **Engine** (`engine.py`) — discrete sim-minute ticks up to the box; each tick every team (in
  sorted id order) advances its current pick by its skill; on solve → points (+ first-blood bonus
  for the global first solver); box expiry → finalize. Ranking: `(points, solves, first_bloods)`
  desc, id asc.
- **Ledger** (`ledger.py`) — every event hash-chained via `ralphcore` (timestamp-free →
  replay-stable). `chain_ok` verifies the chain; `chain_hash` is the run's fingerprint.

## Determinism / replay-stability (verified)

Same `(seed, arena, teams)` → **byte-identical scoreboard AND identical ledger chain hash**. No
wall-clock, no unseeded RNG. The 20-minute "max running time" is the *simulated* box (engine-
enforced); the sim executes in milliseconds.

```
RUN 1 / RUN 2 (seed=1337, box=20min, 4 teams) — identical:
  1. team-3  ratio_balanced  700 pts  3 solves  3 first-blood
  2. team-1  greedy_points   550 pts  1 solves  1 first-blood
  3. team-2  greedy_easy     500 pts  3 solves  1 first-blood
  4. team-4  best_response     0 pts  0 solves  0 first-blood
chain_hash = 6eb937c77fafdbe3dcebabfdc1eab56bdb80fb848be2f0640860a4ab00061978  (both runs)
max solve tick = 20 ≤ box 20   ·   chain_ok = True
ctf replay → "REPLAY OK: 10 events verified; champion team-3 codename praetorian confirmed"
```

## Game theory (`game.py`)

First-blood is a contested resource → challenge selection is a **congestion game**. We analyze
the abstracted one-shot first-pick game (each team picks one challenge; payoff = points + bonus
iff fastest among same-challenge pickers):

- `best_response` strategy maximizes contention-adjusted value (only chases a first-blood it can
  win, else takes guaranteed points).
- `nash_analysis` reports the best-response profile, **`is_nash`** (pure Nash equilibrium of the
  one-shot game — no team profits by unilaterally deviating), per-team **regret**, **social
  welfare**, the **optimum** (brute-forced), and **price of anarchy** (`optimum/welfare ≥ 1.0`).
  Correctness is tested on controlled equilibrium and profitable-deviation cases.
- The demo run reports `nash_equilibrium=True, price_of_anarchy=1.0` (welfare 2050 = optimum).

**Emergent finding:** `best_response` (team-4) scored **0** on the default board — being myopic
per-pick, it over-committed to a high-value/slow target it couldn't finish inside the box, while
the simpler greedy strategies banked points. A genuine game-theoretic outcome: pessimistic
best-response to *contention* ignores the *time budget*.

### v0.1 — box-aware best-response (closes that gap)

`box_aware_response` (`team.py`) extends `best_response` with clock awareness: strategies now
receive an optional `ctx={tick, box_min}`, and box-aware only commits to challenges **finishable
in the remaining box** (`effort/skill ≤ remaining_ticks`), then maximizes the same contention-
adjusted value. It is now the 4th default strategy. Head-to-head on the default board (same seed):

```
team-4  best_response = 0 pts   →   box_aware_response = 550 pts  (last place → 2nd)
```

So the box-aware competitor banks finishable points instead of stalling on an unwinnable target —
the emergent gap, closed by a targeted strategy, verified deterministically. (Existing strategies
ignore `ctx`; with no `ctx`, `box_aware_response` degrades to `best_response`.)

### v0.2 — strategy tournament (which strategy actually wins?)

`tournament.py` (`ctf tournament [--seeds N] [--strategies ...]`) settles it with a deterministic
round-robin sweep: every seed is played under all `n` **cyclic slot-rotations** (each strategy
occupies each slot once per seed → neutralizes the per-team skill bias), aggregating a leaderboard.
Default roster over seeds 1..20 (80 matches/strategy):

```
strategy                 win%   podium%   mean_pts   mean_fb
box_aware_response      56.2%    88.8%      648.8      1.137
ratio_balanced          35.0%    61.2%      529.4      2.513
greedy_points            5.0%    23.8%      301.9      0.287
best_response            3.8%    26.2%      302.5      0.300
```

**At scale, box-aware dominates** (56.2% win / 648.8 pts vs box-blind best_response's 3.8% /
302.5) — the v0.1 thesis confirmed across 80 matches, not one seed. Deterministic (same
seeds+roster → identical leaderboard); one winner per match (Σwins = seeds × rotations).

### v0.3 — season (group stage → knockout bracket → champion)

`season.py` (`ctf season [--group-seeds N] [--bracket-seeds M]`) composes the tournament into a
full calendar: a **group stage** (the round-robin sweep) ranks the field, then seeds a
**single-elimination bracket** (1v4, 2v3 → final for 4 competitors; top-2 final otherwise), each
bracket match being a head-to-head sub-tournament whose higher win-rate advances. Deterministic
default season:

```
group: box_aware_response(1) · ratio_balanced(2) · greedy_points(3) · best_response(4)
SF1  box_aware_response vs best_response → box_aware_response   (1 v 4)
SF2  ratio_balanced     vs greedy_points  → ratio_balanced       (2 v 3)
final  box_aware_response vs ratio_balanced → box_aware_response
🏆 season champion: box_aware_response   (runner-up: ratio_balanced)
```

Same `(strategies, group_seeds, bracket_seeds, box)` → identical bracket + champion.

### v0.4 — game types: king-of-the-hill (`koth`)

The arena now has a `mode` (`jeopardy` default, or `koth`); `ctf run --mode koth`. **koth** =
king-of-the-hill: solving a challenge makes you its **holder**; each subsequent tick the holder
earns the challenge's `points`; a rival can **re-solve to steal** the hold. Score accrues over
time → a contested, time-dynamic game (`engine._run_koth`; the jeopardy path is untouched).
Deterministic default koth run:

```
1. ratio_balanced      3300 pts  5 captures      ← holding HIGH-value challenges, not most captures
2. box_aware_response  2100 pts  2 captures
3. greedy_points       2000 pts  1 capture
4. greedy_easy          700 pts  4 captures      ← captured cheap challenges → low hold income
(12 captures, 7 of them steals from a prior holder)
```

**The game type changes the meta:** `ratio_balanced` wins koth (hold high points/effort longest),
whereas `box_aware_response` wins jeopardy — different rules reward different strategies. Same
`(seed, arena, teams)` → identical scoreboard + chain hash; captures (incl. steals) are
hash-chained `capture` events; the box is enforced.

## Reward — non-currency (`reward.py`)

The champion is **not** paid currency. They earn a **codename honorific + a verifiable trophy
attestation**: a lore codename (`praetorian`, `brett-shaw`, …) derived deterministically from the
run's pre-trophy chain hash, minted as the **final chained ledger entry**. So the trophy is
tamper-evident and a third party can confirm `champion` earned `codename` for the run whose hash
is `bound_to` — recognition + a cryptographic bragging right, replay-stable.

## Scope / follow-ups

**Dogfood:** `vaked/examples/ctf-arena.vaked` declares the competitor roster as a POLA capability
graph (a referee delegates strictly attenuated, read-only authority to each team) — passes
`vakedc check` + lowers, mirroring `redteam-swarm.vaked` / `oracle-team.vaked`.

Deterministic strategy sim only. Out of scope (future cycles): agent/LLM-driven teams (real
solving), live challenge infra, inter-team sabotage/steals, a box-aware `best_response`, and a TUI
scoreboard surface.
