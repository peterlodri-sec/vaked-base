WP4-S2 — sandboxd isolation backend: namespaces + cgroups v2 + seccomp-bpf (native-exec, pure Zig)

Status

Implementation spec (2026-06-14). Sprint WP4-S2 of the WP4 daemon-MVP track
(docs/superpowers/plans/2026-06-14-wp4-kickoff.md,
table row "WP4-S2 | Jul 9–Jul 23 | sandboxd: isolation backend (namespaces +
cgroups)"). This is the implementation of Plan step 2 ("native-exec for
real") of the sandboxd design
(docs/superpowers/specs/2026-06-13-sandboxd-design.md),
extended with the seccomp-bpf allowlist that the task title and kickoff line
44 (CLONE_NEWNS | CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWUSER + seccomp-bpf
allowlist) place in this backend.

Scope reconciliation (named, not silently resolved): the kickoff table
labels S2 "namespaces + cgroups" and puts "capability enforcement hooks" in S5;
the kickoff decision/first-task prose (lines 44, 60–61) and this task's title
both name seccomp as part of native-exec. This spec follows the task + the
design's native-exec mechanics §step 5 ("apply the seccomp / no-new-privs
posture, then execve"): the seccomp filter install is S2; the
capability→syscall-set lowering (which grant maps to which allowlist) is the
S5 hook. S2 ships a fixed, conservative allowlist; S5 makes it grant-driven.

Build target is dev-cx53 (Linux x86_64, Nix 2.34.7, kernel 6.x, cgroup-v2;
kickoff lines 21–22). Min Zig 0.16 (verified on this host: zig version →
0.16.0). dev-cx53 is OFF-LIMITS for the current autoresearch window;
everything below marked M1-local must pass on aarch64-darwin without it.

1. Objective

Replace the WP4-S1 no-op/echo runner in daemons/sandboxd/ with a real
native-exec isolation boundary: per worker, sandboxd

  1. enters a user + mount + pid + net namespace set
    (CLONE_NEWUSER | CLONE_NEWNS | CLONE_NEWPID | CLONE_NEWNET),
  2. joins a delegated cgroup-v2 leaf and writes kernel-enforced hard limits
    (memory.max, pids.max, cpu.max, io.max),
  3. assembles an overlayfs rootfs (ro lowerdir + writable upperdir/workdir)
    and explicit bind-mounts only (deny-by-default fs view),
  4. installs a seccomp-bpf allowlist under PR_SET_NO_NEW_PRIVS,
  5. drops privileges and execves the workload, then supervises the child
    (reap, report exit) and appends spawn/kill/exit audit events to eventd.

The bound is the kernel's (OOM-kill, clone refusal, seccomp KILL/ERRNO,
absent mounts) — "mathematical bound, not policy" (design §native-exec
mechanics, §Security model). The Python package
agent_sandboxd/ is the behavioural oracle for every
module except seccomp (which the reference never implemented — see §3).

1.1 In scope

  • The five mechanics above, in pure Zig syscall wrappers (no libc sandbox lib,
    no libseccomp, no runc/OCI; matches the empty dependencies in
    daemons/sandboxd/build.zig.zon).
  • A test suite split into M1-local pure-logic assertions and dev-cx53 kernel
    effects (§5).

1.2 Out of scope (explicit, with provenance so a reader does not think they were missed)

  • S1 skeleton — CLI flag parsing, gen/zig/<fiber>.json parse, the
    no-op runner shell. Assumed to exist from WP4-S1 (kickoff "First task"); this
    spec extends it. (Today only the stub daemons/sandboxd/src/main.zig exists;
    if S1 has not landed, its deliverables are a hard prerequisite — see §6.)
  • Config projection (design Plan step 0, grammar-first, gated) — lowering
    capability fs/capability process grants + a new OS-resource-limits field
    to gen/zig/<fiber>.json, resolving engine_package to a binary+argv, and
    routing an agent-workload fiber's config to sandboxd. This is a blocking
    dependency    , not S2 work (design §Config contract, §Open). S2 runs against
    hand-written fixture configs (§2.2) until step 0 lands.
  • S3 supervisord→sandboxd port (design Plan step 3, "Zig-port era") — the
    vaked_fiber_worker port and OTP lifecycle wiring. S2 appends to the existing
    eventd log; it does not build the supervisord port. No second lifecycle: the
    STOPPED→…→RUNNING transitions are owned by agent-supervisord
    (RFC 0004 §6, which
    mandates supervisord rejects RUNNING until cold-start verification passes; and
    RFC 0001 §5 "Supervision and state
    dependencies: agent-supervisord") — sandboxd is only the exec mechanism the
    worker is run through (design §Relationship to the roster). S2 starts/stops the
    process; it does not transition agent state.
  • fs-snapshotd accounting — diffs / write-budgets / artifact capture over the
    upperdir. S2 mounts the overlay; it does not account (design §Relationship
    to the roster).
  • No canonical hashing — the eventd oracle owns bytes; sandboxd emits payloads
    only (design §What is deliberately NOT here;
    agent_sandboxd/eventd.py docstring).
  • netns programming — sandboxd creates the netns; agent-guardd programs its
    firewall (design §Open "Network-namespace ownership";
    agent_sandboxd/namespace.py lines 19–23).

2. Inputs and oracles

2.1 Oracles (per-module Python reference)

Each Zig module's correctness oracle is its 1:1 Python counterpart in
agent_sandboxd/. Zig modules must reproduce the oracle's decisions and
emitted bytes/strings, not its mechanism (the reference uses the unshare(1)
CLI and writes files; Zig uses raw syscalls — design §native-exec mechanics
notes this divergence explicitly).

Zig module (to create) Python oracle What is held identical
config.zig (S1) + agent_sandboxd/daemon.py _parse_spawn_request field set + defaults of the spawn request
namespace.zig namespace.py build_namespace_plan the unshare/clone flag set, mount step list, overlay opts string, uid/gid map
cgroup.zig cgroup.py apply_cgroup_limits exact file contents (memory.max, pids.max, cpu.max = "<quota> <period>"/"max <period>", io.max per-device line) and leaf path vaked/<agent-id>/
policy.zig policy.py decide_path deny-by-default verdict + reason per (path, mode); fs lattice none<repo_ro<repo_rw<host_rw, repo_ro<host_ro
event.zig eventd.py spawn_event/kill_event/access_event payload kind/v/field set; no wall-time in the chained body
seccomp.zig none (see §3) modelled on agent_guardd/bpf.py ABI pattern

3. The seccomp asymmetry

seccomp.zig is the one module with no Python oracle — the reference
agent_sandboxd never implemented seccomp (grep confirms: seccomp appears only
in design/README/threat-model prose, not in agent_sandboxd/*.py). Its model is
the ABI-driven pattern of agent_guardd/bpf.py:
assemble the BPF program as struct-packed instructions directly (no
libseccomp, no libbpf — guardd drives bpf() with raw structs and
architecture-neutral bytecode; seccomp.zig does the same for the classic-BPF
filter). This is the runtime defense the threat model assigns to the daemon layer
(docs/language/THREAT_MODEL.md §Scenario C
item 1: "Zig daemons should run in OS-level sandboxes (seccomp, …)").

Because there is no byte-oracle, the seccomp acceptance bar is self-consistent
byte assertions (the filter bytes equal a hand-derived golden, §5.1) plus the
kernel behaviour (a disallowed syscall is killed/ERRNO'd, §5.2).

5. Algorithm / design

5.1 Per-worker spawn sequence

exec.zig: spawn(cfg) runs, in order (mirrors design §native-exec mechanics
1→5 and daemon.py: SandboxDaemon.spawn):

spawn(cfg):
  0. policy gate    — if cfg.process_policy.grant == none: write deny event, return Denied
                      (reference phase: allow-all + log otherwise — design §Security model;
                       daemon.py lines 149–170. Authority ultimately belongs to preceptord,
                       RFC 0001 §5 "Authority and policy: preceptord" — the authority plane that
                       checks a peer may access the process/filesystem capability; absent in the
                       reference phase, so the grant check is stubbed-but-wired here.)
  1. write-ahead    — event.spawn(agent_id, argv, fs_grant, proc_grant, action="allow") → eventd
  2. cgroup leaf    — cgroup.create_leaf(agent_id); cgroup.write_limits(cfg.limits)
                      (BEFORE clone, so the child is born already accounted — write the child
                       pid into cgroup.procs after clone, before it execs)
  3. clone child    — clone(CLONE_NEWUSER|CLONE_NEWNS|CLONE_NEWPID|CLONE_NEWNET|SIGCHLD)
                      parent: record host-view pid; child: continue at step 4
  4. (child) userns — write /proc/self/uid_map = cfg.uid_map, gid_map = cfg.gid_map,
                      setgroups=deny first (kernel requires it before gid_map for unpriv)
  5. (child) mountns— mount("","/","",MS_REC|MS_PRIVATE); assemble overlay rootfs
                      (mount.zig: overlay lowerdir/upperdir/workdir), bind cfg.binds
                      (ro binds get a second MS_REMOUNT|MS_RDONLY pass), then pivot_root
                      into the new rootfs; unmount the old root (deny-by-default fs view)
  6. (child) cgjoin — already joined via parent writing pid to cgroup.procs (step 2 note)
  7. (child) seccomp— prctl(PR_SET_NO_NEW_PRIVS,1); seccomp(SECCOMP_SET_MODE_FILTER, filter)
                      (filter built by seccomp.zig — step 5.3). MUST be after no_new_privs,
                      BEFORE drop-privs and execve.
  8. (child) drop   — setresgid/setresuid to the unprivileged target (root only inside userns)
  9. (child) exec   — execve(cfg.exec.path, argv, env)
 10. (parent) reap  — waitpid; on exit write event.exit(agent_id, pid, code) → eventd

Ordering invariants that make this implementation-ready:
setgroups=deny before gid_map; PR_SET_NO_NEW_PRIVS before
SECCOMP_SET_MODE_FILTER; seccomp install after all privileged setup
(mounts, cgroup, uid_map) and before drop-privs+execve (so the filter
covers the workload but not sandboxd's own setup syscalls); cgroup limits written
before the child can fork (so pids.max is enforced from pid 1).

5.2 namespace.zig / mount.zig

  • build_plan(cfg) (pure, M1-testable) produces the same logical plan as
    namespace.py: build_namespace_plan: the CLONE_* flag mask, the ordered
    bind-mount list, the overlay options string
    lowerdir=<l>,upperdir=<u>,workdir=<w>, the uid/gid map lines. This is the
    unit under M1 byte/string assertions; it performs no syscalls.
  • enter(plan) (effectful, dev-cx53) issues the syscalls. Divergence from the
    reference: the reference uses overlay mounted at / then no pivot; the Zig
    impl mounts overlay into a staging dir and pivot_roots (design line 189
    "the Zig daemon will construct an overlay rootfs + pivot_root instead").

5.3 seccomp.zig — the allowlist filter

Classic-BPF program (the format seccomp(SECCOMP_SET_MODE_FILTER) consumes),
assembled as a []sock_filter then wrapped in sock_fprog. Structure:

# 1. arch gate: load seccomp_data.arch; if != AUDIT_ARCH_X86_64 → RET_KILL_PROCESS
# 2. load seccomp_data.nr (syscall number)
# 3. for each nr in ALLOWLIST: jeq nr → RET_ALLOW
# 4. fallthrough: RET_ERRNO(EPERM)   (default-deny; KILL_PROCESS is the harder variant)

S2 ships one fixed default profile (a conservative allowlist sufficient for
a typical compute worker: read write openat close fstat lseek mmap munmap mprotect brk rt_sigaction rt_sigprocmask rt_sigreturn ioctl(restricted) clock_gettime exit exit_group nanosleep futex getrandom pread64 pwrite64 … —
the exact set is fixed in seccomp.default.golden). The default action is
SECCOMP_RET_ERRNO | EPERM (fail-closed but observable; RET_KILL_PROCESS is a
config toggle for the strict profile). Grant-driven profile selection
(spawn_sandboxed ⊂ spawn ⊂ exec_host ⇒ progressively wider allowlists) is
the S5 hook, not S2; S2 leaves the selection point stubbed-but-wired (the
cfg.seccomp.profile field is read and routed; only "default" resolves today).

Assembly is pure — assemble(profile) []sock_filter does no syscalls and is
M1-tested by asserting the byte array equals seccomp.default.golden (the
guardd-_insn precedent: assert exact instruction bytes). install(filter) is
the effectful prctl+seccomp pair, dev-cx53 only.

5.4 cgroup.zig

Pure helpers (leaf_path(agent_id) = "vaked/<agent-id>",
format_cpu_max(quota, period), format_io_max(dev)) plus effectful writers.
The pure formatters reproduce cgroup.py's exact strings; the writers take a
cgroup_root parameter so M1 tests redirect to a temp dir (the cgroup_root
mock discipline of cgroup.py §Mock path). Teardown rmdirs the leaf after the
last process exits (kernel refuses otherwise — cgroup.py teardown_cgroup).

Oracle note: the parity boundary is apply_cgroup_limits (same input → same file
contents), not daemon.py: SandboxDaemon.spawn. daemon.spawn hardcodes
cpu_quota_us=None (writing "max <period>") and defers the cpu_weight→cpu.max
wiring as an impl detail; the §2.2 fixture supplies a concrete cpu_quota_us, so
format_cpu_max is tested against the formatter directly. A reader comparing to
daemon.spawn's call site will see that omission — it is intentional, not a
discrepancy.

5.5 policy.zig

Port of decide_path: deny-by-default prefix match (commonpath-equivalent via
normalised-path prefix check), write ⇒ only allowed_write, read ⇒
allowed_read ∪ allowed_write, grant == none ⇒ always deny. Pure; M1-tested
against the same table the reference uses (__main__.py: cmd_demo cases:
/repo/out write allow, /repo read allow, /etc/passwd read deny).

8. Acceptance criteria

S2 is done when all hold:

  1. Pure-logic parity (M1-local). config.zig, namespace.zig (plan),
    cgroup.zig (formatters + content), policy.zig, event.zig produce
    outputs identical to their Python oracles (§2.1) for the §2.2 fixture and the
    agent_sandboxd demo cases. zig build test is green on aarch64-darwin.
  2. seccomp filter assembly (M1-local). seccomp.zig: assemble("default")
    equals testdata/seccomp.default.golden byte-for-byte; the filter encodes
    arch-gate (AUDIT_ARCH_X86_64 mismatch → KILL), the fixed allowlist
    (each → RET_ALLOW), and default RET_ERRNO(EPERM) (§5.3).
  3. Cross-compile (M1-local). zig build -Dtarget=x86_64-linux compiles the
    full daemon (no syscall executed on M1).
  4. Boundary holds (dev-cx53). task sandbox-smoke asserts: child is pid 1 in
    its pid ns and host mounts are absent; over-memory.max child is OOM-killed
    and a pids.max breach refuses clone; the overlay is the rootfs and writes
    land on upperdir; an undeclared host path is absent; a syscall outside the
    allowlist is ERRNO'd/killed     (the S2-specific addition).
  5. Audit (dev-cx53). A real spawn writes write-ahead sandbox_spawn
    (action allow/deny), and exit writes a sandbox_exit/sandbox_kill
    payload to eventd; grant == none is refused and logged action="deny"
    (daemon.py parity). No canonical hashing in sandboxd (eventd owns bytes).
  6. Ordering invariants (dev-cx53, asserted by the smoke harness where
    observable).     setgroups=deny precedes gid_map; no_new_privs precedes
    the seccomp filter; the filter is installed after privileged setup and before
