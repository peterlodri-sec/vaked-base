Owner-approved scope.</p> <h2 id="purpose">Purpose</h2> <p>An always-on "Ralph loop" that keeps <strong>at least one</strong> decision agent running,<br /> <strong>round-robining</strong> across the fleet's repos. Each iteration reads a repo's<br /> current state + its prior decisions, surfaces (or advances) the single most<br /> important open <strong>strategic decision</strong> for that repo, and appends a structured,<br /> dated entry to that repo's <strong>advisory</strong> decisions log (human-ratified). It runs<br /> on cheap MoE models via OpenRouter, and an <strong>eye dashboard</strong> (terminal live<br /> view) shows what's running, what it decided, and what it has spent.</p> <blockquote> <p>"Ralph" = re-invoke an agent with a fixed task on a loop, persisting state<br /> (the decisions log) between iterations so it makes incremental progress.</p> </blockquote> <h2 id="targets-config-driven-extensible">Targets (config-driven, extensible)</h2> <p>A repo list — start with three, add agentfield components one by one later:</p> <table> <thead> <tr> <th>name</th> <th>path</th> <th>gh repo</th> </tr> </thead> <tbody> <tr> <td><code>vaked-base</code></td> <td><code>~/workspace/peterlodri-sec/vaked-base</code></td> <td><code>peterlodri-sec/vaked-base</code></td> </tr> <tr> <td><code>crabcc</code></td> <td><code>~/workspace/peterlodri-sec/crabcc</code></td> <td><code>crabcc-labs/crabcc</code></td> </tr> <tr> <td><code>agentfield-stack</code></td> <td><code>~/workspace/peterlodri-sec/agentfield-stack</code></td> <td><code>peterlodri-sec/agentfield-stack</code></td> </tr> </tbody> </table> <p>Config lives in <code>tools/ralph/repos.json</code> (name, path, gh, optional <code>subdir</code> for<br /> later per-agent/module targets). Adding a target = one JSON entry.</p> <h2 id="non-goals">Non-goals</h2> <ul> <li><strong>Not</strong> a code-writing agent. Read-only on all target repos except <em>appending</em><br /> to decision logs (which live in the <strong>home</strong> repo, see State).</li> <li><strong>Not</strong> a Claude Code loop — standalone, talks to OpenRouter directly.</li> <li>No tool-calling, no git writes, no network beyond OpenRouter (+ local<br /> <code>gh</code>/<code>git</code>/file <em>reads</em> per target).</li> </ul> <h2 id="components">Components</h2> <p>One stdlib-only script, <code>tools/ralph/ralph.py</code> (no pip deps; <code>urllib</code> for HTTP),<br /> with three subcommands — each a small, independently-testable unit:</p> <ol> <li><strong><code>ralph decide --repo <name></code></strong> — run <strong>one iteration</strong> for one repo (the<br /> core, two-stage; below). Pure-ish: reads state, calls OpenRouter, appends one<br /> log entry, returns a result record (iteration #, decision title, cost).</li> <li><strong><code>ralph run</code></strong> — the <strong>supervisor</strong>: a continuous round-robin that keeps<br /> exactly one loop alive at a time, cycling the repo list, pacing with<br /> <code>--interval</code>, and stopping only on the cumulative <strong>budget backstop</strong> or<br /> Ctrl-C. Writes <code>status.json</code> after every step.</li> <li><strong><code>ralph watch</code></strong> — the <strong>eye dashboard</strong>: a refreshing terminal table read<br /> from <code>status.json</code> + log tails. No browser.</li> </ol> <h2 id="iteration-two-stages-two-models-ralph-decide">Iteration — two stages, two models (<code>ralph decide</code>)</h2> <h3 id="stage-1-surface-rank-qwenqwen3-235b-a22b-thinking-2507">Stage 1 — surface & rank (<code>qwen/qwen3-235b-a22b-thinking-2507</code>)</h3> <p>Reads a <strong>compact</strong> state for the target repo (open-issue titles+bodies, recent<br /> <code>git log</code>, short digests of key docs, and the <strong>titles</strong> of that repo's prior log<br /> entries) → a ranked candidate list. The thinking MoE does the reasoning-heavy<br /> "what matters most now" step. Request body:</p> <ul> <li><code>model</code>, <code>messages</code></li> <li><code>reasoning</code>: <code>{ "enabled": true, "effort": "medium" }</code></li> <li><code>response_format</code>: JSON schema →<br /> <code>{ "candidates": [ { "title", "why_now", "urgency": 1-5, "addressed": bool } ] }</code></li> <li><code>temperature</code> 0.4, <code>top_p</code> 0.95, <code>seed</code> (<code>--seed</code>, default 42), <code>max_tokens</code> ~2000</li> </ul> <p><strong>Selection (in-script, deterministic):</strong> highest-<code>urgency</code> <code>addressed=false</code><br /> (ties → first); if all addressed, highest-<code>urgency</code> to advance with new info.</p> <h3 id="stage-2-deep-dive-deepseekdeepseek-v4-flash-1m-ctx">Stage 2 — deep-dive (<code>deepseek/deepseek-v4-flash</code>, 1M ctx)</h3> <p>Given the chosen candidate <strong>+ the full state for that repo</strong> (full issue bodies,<br /> full <code>git log</code> window, full doc text, that repo's <strong>entire</strong> prior log — the 1M<br /> window holds it) → the detailed entry. <code>temperature</code> 0.3, <code>top_p</code> 0.95,<br /> <code>max_tokens</code> ~1800. No <code>reasoning</code>/<code>tools</code>.</p> <h2 id="state-all-in-the-home-repo-read-only-on-the-others">State (all in the HOME repo — read-only on the others)</h2> <p>The tool's home is <code>vaked-base</code>. <strong>No writes to crabcc/agentfield-stack</strong> — their<br /> decision logs live here:</p> <ul> <li><code>docs/decisions/<repo>.ralph-log.md</code> — one advisory log per target repo;<br /> header marks it machine-generated; entries <strong>appended, never rewritten</strong>.</li> <li><code>tools/ralph/state/status.json</code> — the live supervisor state the dashboard reads<br /> (current repo, iteration, last decision title per repo, per-repo + total spend,<br /> started-at, last-step-at, status, next repo).</li> </ul> <p>Entry format (per log):</p> <div class="highlight"><pre><span></span><span class="gu">##</span> 2026-06-11 — Decision #N: <title> <span class="k">-</span> **Repo:** crabcc · **Models:** stage1 qwen3-235b-a22b-thinking · stage2 deepseek-v4-flash <span class="k">-</span> **Context snapshot:** HEAD <sha>, <K> open issues <span class="k">-</span> **Decision / question:** … <span class="k">-</span> **Options:** … **Recommendation:** … **Risks:** … <span class="k">-</span> **Next actions:** … **Confidence:** low|med|high </pre></div> <p>Continuity: prior entry <em>titles</em> feed Stage 1 (avoid repeats); the <em>full</em> per-repo<br /> log feeds Stage 2.</p> <h2 id="supervisor-ralph-run-always-1-running">Supervisor (<code>ralph run</code>) — "always ≥ 1 running"</h2> <p>Continuous loop: pick next repo (round-robin) → <code>decide</code> one iteration → update<br /> <code>status.json</code> → <code>sleep(--interval)</code> → repeat. Stops only on the cumulative<br /> <strong>budget backstop</strong> (<code>--budget-total</code>) or SIGINT (clean shutdown writes a final<br /> status). Pacing (<code>--interval</code>, default 900s/15min) keeps spend low and gives<br /> humans time to ratify. Exactly one iteration runs at a time (no concurrency).</p> <p>Backstops (non-bypassable, checked before each call): cumulative USD cap<br /> (<code>--budget-total</code>, default $2.00) and an optional max-iterations<br /> (<code>--max-iters</code>, default unbounded for <code>run</code>). On restart it resumes spend +<br /> round-robin position from <code>status.json</code>.</p> <h2 id="eye-dashboard-ralph-watch">Eye dashboard (<code>ralph watch</code>)</h2> <p>A <code>watch</code>-style terminal view (redraw every <code>--refresh</code>, default 3s) reading<br /> <code>status.json</code> + the tail of each log. Columns/sections:</p> <ul> <li><strong>Loop:</strong> ● running / ○ idle, current repo, iteration #, seconds since last step.</li> <li><strong>Per repo:</strong> last decision title, entry count, last-run time, spend.</li> <li><strong>Spend:</strong> total USD vs <code>--budget-total</code> (a simple bar), today's count.</li> <li><strong>Recent decisions:</strong> last ~5 titles across repos with repo + timestamp.</li> </ul> <p>ASCII only (no deps). If <code>status.json</code> is missing/stale → "no supervisor running"<br /> (this is how you see whether the "always-on" loop is actually alive).</p> <h2 id="budget-accounting">Budget & accounting</h2> <p>Each response carries <code>usage</code>. Cost/call = <code>prompt_tok × in_price + completion_tok × out_price</code>, per-model (defaults qwen <code>$0.10/$0.10</code>,<br /> deepseek-v4-flash <code>$0.098/$0.197</code> per 1M — fetched from OpenRouter <code>/models</code> at<br /> startup, hardcoded fallback with a "refresh" note). Cumulative total persists in<br /> <code>status.json</code>; checked before every call.</p> <h2 id="cli">CLI</h2> <div class="highlight"><pre><span></span><span class="n">tools</span><span estimate; no API<br /> call, no log write.</p> <h2 id="privacy-endpoint-data-governance">Privacy / endpoint (data governance)</h2> <p>A live iteration sends <strong>private-repo content</strong> (issue bodies, README/CLAUDE.md,<br /> git log) to whatever endpoint is configured. By default that is <strong>OpenRouter</strong>, a<br /> third party outside the trust boundary — accept this consciously, or override the<br /> endpoint. Precedence: <code>--base-url</code> > <code>RALPH_BASE_URL</code> env > OpenRouter default;<br /> key: <code>RALPH_API_KEY</code> > <code>OPENROUTER_API_KEY</code>. Point <code>--base-url</code> /<code>RALPH_BASE_URL</code><br /> at a <strong>self-hosted, trust-boundary OpenAI-compatible endpoint</strong> (e.g.<br /> <code>agentfield-inference-host</code>) to keep private content local. (A future<br /> <code>--titles-only</code> context mode could further reduce what leaves the box.)</p> <h2 id="safety">Safety</h2> <ul> <li>Read-only on every target repo; the only writes are appends to logs +<br /> <code>status.json</code>, both under <code>vaked-base</code>.</li> <li>API key (<code>OPENROUTER_API_KEY</code>) from env only; never logged.</li> <li>Budget + (optional) iteration caps are non-bypassable backstops.</li> <li>Network: only <code>https://openrouter.ai</code>; context is local <code>gh</code>/<code>git</code>/file reads.</li> <li><code>gh</code> against private repos (crabcc) uses the existing <code>gh</code> auth.</li> </ul> <h2 id="error-handling">Error handling</h2> <ul> <li>Missing <code>OPENROUTER_API_KEY</code> → clear error, exit non-zero.</li> <li>OpenRouter non-200 / rate-limit → bounded retry+backoff (3 tries), then skip<br /> that iteration (note to status + stderr), supervisor continues to next repo.</li> <li>Stage-1 JSON parse failure → one reformat retry; else skip iteration (warn).</li> <li>A target repo path/<code>gh</code> unavailable → skip that repo this cycle, note it;<br /> supervisor keeps the others running ("always ≥1").</li> <li>SIGINT → finish the in-flight call's accounting, write final status, exit 0.</li> </ul> <h2 id="testing">Testing</h2> <ul> <li><strong><code>decide --dry-run</code></strong> smoke (no network): builds prompts, prints estimate,<br /> writes nothing.</li> <li><strong>Selection unit test</strong>: fixed Stage-1 JSON → asserts highest-urgency-unaddressed<br /> pick + all-addressed fallback.</li> <li><strong>Cost-math test</strong>: known <code>usage</code> + price → expected USD (pure).</li> <li><strong>Round-robin test</strong>: next-repo advances and wraps; skips an unavailable repo.</li> <li><strong>Budget backstop test</strong>: <code>--budget-total 0</code> → zero calls, clean exit.</li> <li><strong>status.json round-trip</strong>: supervisor writes → <code>watch</code> parses (no live API).</li> </ul> <p>(The two network stages aren't unit-tested live; the pure pieces are.)</p> <h2 id="files">Files</h2> <table> <thead> <tr> <th>File</th> <th>Role</th> </tr> </thead> <tbody> <tr> <td><code>tools/ralph/ralph.py</code></td> <td>the script (decide / run / watch), stdlib-only</td> </tr> <tr> <td><code>tools/ralph/repos.json</code></td> <td>target repo list (extensible)</td> </tr> <tr> <td><code>tools/ralph/state/status.json</code></td> <td>live supervisor state (gitignored)</td> </tr> <tr> <td><code>docs/decisions/<repo>.ralph-log.md</code></td> <td>per-repo advisory decision logs</td> </tr> <tr> <td><code>tests/</code></td> <td>selection · cost-math · round-robin · dry-run · status round-trip</td> </tr> </tbody> </table> <h2 id="build-order-phases">Build order (phases)</h2> <ol> <li><strong><code>decide</code></strong> (one repo, two-stage, log append, <code>--dry-run</code>) + its unit tests.</li> <li><strong><code>run</code></strong> supervisor (round-robin, budget backstop, <code>status.json</code>, pacing).</li> <li><strong><code>watch</code></strong> dashboard.</li> <li>Grow <code>repos.json</code> with agentfield-stack components one at a time.</li> </ol> <h2 id="cost-envelope-sanity">Cost envelope (sanity)</h2> <p>~10–20k in + ~3k out/iteration across both models ≈ <strong>$0.002–0.005/iteration</strong>.<br /> At <code>--interval 900</code> (4/hr) that's well under a cent/hour per repo; the<br /> <code>--budget-total</code> cap (default $2.00) is the real ceiling and bounds a runaway.</p> <div class="footer"> <span><a href="https://github.com/peterlodri-sec/vaked-base/blob/main//Users/lodripeter/workspace/peterlodri-sec/vaked-base/docs/superpowers/specs/2026-06-11-ralph-decide-design.md" style="color:var(--text2);font-size:.78rem">View source →</a></span> <span>Genesis seal <code style="font-size:.7rem">7c242080</code></span> </div> </div> </div> </body> </html>
