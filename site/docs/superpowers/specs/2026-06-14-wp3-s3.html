0. Citation correction (read first)

The WP3 kickoff (docs/superpowers/plans/2026-06-14-wp3-kickoff.md line 43) and
the S3 issue text say golden vectors come from "RFC 0002 §4 / Appendix A".
That wording is stale and wrong:

  • The byte encoding lives in RFC 0002 §6 ("hcpbin encoding rules"), not §4.
  • The frame header (kind/corr/stream/seq/end) is RFC 0002 §4.2, but
    its wire encoding is normative in RFC 0003 §4.4.1, and the header is part of
    the WIRE layer, explicitly not part of hcpbin's payload scope
    (RFC 0002 §4.2; RFC 0003 §4.4).
  • There is no Appendix A in RFC 0002. Golden vectors are the worked examples
    in RFC 0002 §6.1.1, §6.3.1, §6.6.1, §6.7.1, and §10.

All citations below use the corrected section numbers. RFC 0004 is referred to by
the repo path protocol/rfcs/0004-multi-agent-state-dependency.md (its abstract
notes it was informally called "RFC 0005" in pre-2026-06-12 design sessions; that
alias is dead — RFC 0005 is the unrelated control-frame catalog).

1. Objective

Deliver the message-routing layer of the litany crate: the component that
takes an already-decoded inbound Votive Frame and decides where it goes —
demultiplexing by frame header, reassembling streams, matching responses to
outstanding requests, binding frames to chapters, and gating delivery on the
RFC 0004 state-dependency model (topology-epoch currency and consumer lifecycle
state). It produces, for each inbound frame, exactly one routing verdict:

Deliver(target, frame)        # hand to the destination handler / mailbox (Request, Control)
Reply(corr, frame)            # match to an outstanding local exchange by corr
                              #   (Response, Error: terminal; Event: to the subscription handler)
Queue(target, frame)          # buffer: target is paused (RFC 0003 §2.1 / RFC 0005 §2.1)
Refuse(corr, error_kind)      # mint a frame-level `error` (RFC 0003 §9.2)
Reassemble(stream, ...)       # accumulate a stream chunk; not yet terminal

1.1 What this sprint is NOT (scope boundary — normative for the design)

The router consults state-dependency state; it does not own or enforce
RFC 0004's machinery. The following are explicitly out of scope for S3 and
belong to other components:

Concern Owner RFC
GC-floor computation, checkpoint leases, eviction eventd + agent-supervisord (WP3-S5, supervisor) RFC 0004 §4
Cold-start verification execution (the verdict the supervisor acts on) agent-supervisord over eventd RFC 0004 §6
The dependency index / anchor freshness source of truth eventd (eventd/statedep.py ref → Zig port) RFC 0004 §8 order 4
DAG validation vakedc compiler pass RFC 0004 §5
Control-frame catalog (PauseControl, RewindControl, …) semantics + its refusals WP3-S4 RFC 0005
Transport (stdio/UDS/vsock byte-streams), handshake, framing bytes WP3-S2 (below the router) RFC 0003 §3–§5

The router's RFC-0004 surface is one routing gate plus one read-only seam:

  1. Lifecycle gate (the genuinely-routing RFC 0004 obligation) (RFC 0004 §6):
    a target in PAUSED(stale_dependency) MUST NOT receive normal work frames;
    they are queued (or refused) until the supervisor clears the pause. The router
    reads the lifecycle state from an injected source (TargetState); it never
    performs the verification that sets that state.
  2. Epoch source (read-only, NOT a router-minted refusal) (RFC 0004 §7 / §7.1):
    the router reads the current topology epoch from an injected EpochSource so it
    can pass it to target resolution (RFC 0005 §2.4). It does not mint a
    stale_epoch wire error: a frame with a stale topology_epoch is delivered to
    S4/the supervisor    , which validates the epoch and returns an RFC 0005
    ControlAck{reason=stale_epoch} (control frames) or re-verifies against the old
    epoch's graph (state-dependency frames, RFC 0004 §7.1). Epochs are
    supervisor-assigned; the router never asserts, mutates, or refuses on them.

The lifecycle gate is the only RFC 0004 verdict the router mints; the epoch is a
read-only input it forwards.

If S3 reimplements compute_gc_floor or verify_cold_start inside the litany
crate, it has accidentally written the S5 / supervisor spec. The router's job is
demux + the one read-only lifecycle gate.

2. Inputs / oracles

2.1 Normative inputs (RFCs)

  • RFC 0002 §4.2 — frame header fields (kind/corr/stream/seq/end);
    reserved @0 tag space; author tags begin at @1.
  • RFC 0002 §4.3 — the five frame classes via service: call (Req)->Resp
    binds request↔response; subscribe (Ctrl) yields Ev binds a control frame to a
    terminated event stream (end=true). Authority is service/class/digest-scoped.
  • RFC 0003 §4.4.1 — frame header wire encoding (kind varint 0–4, corr
    16-byte UUID, optional stream/seq varints, end byte; decoder invariant and
    default-omission rules). This is the byte layout the router's FrameHeader
    mirrors; the decode itself is S2's frame layer.
  • RFC 0003 §6.1–6.3 — connection/chapter/stream scopes; multiplexing by
    corr/stream/seq/end; initiator-even / responder-odd stream-id parity;
    end retires a stream id (reuse forbidden); seq gap ⇒ frame-level error.
  • RFC 0003 §7.2 — control-class frames take the priority path exempt from
    flow control    ; the router MUST NOT queue control frames behind data backpressure.
  • RFC 0003 §9.2 / §9.2.1 — frame-level error taxonomy: malformed_frame,
    unknown_service, unknown_method, authority_denied, seq_gap,
    chapter_closed, stream_reset, invalid_correlation (the normative set lives
    in the hcp.wire schema, RFC 0003 Appendix A).
  • RFC 0004 §6 — cold-start lifecycle states and PAUSED(stale_dependency);
    the verdict the lifecycle gate honors.
  • RFC 0004 §7 / §7.1 — topology epoch; immutable per chapter (set at open,
    RFC 0003 §6.2); stale-epoch frames refused or re-verified; epoch is
    supervisor-assigned, never caller-asserted.
  • RFC 0005 §2.4 / §6 — (target, topology_epoch) name resolution; the
    unknown_target / stale_epoch refusals are RFC 0005 ControlRefusal reasons
    carried in a ControlAck (response-class frame), not RFC 0003 wire
    error_kinds — they are minted by S4/the supervisor, not the router. (RFC 0005
    is S4; S3 references the resolution contract via the TargetResolver seam, §3.3,
    but does not implement the control catalog or its refusals.)

2.2 Differential oracle (executable, runs on M1)

The Python reference of the RFC 0004 state-dependency layer is the oracle for
the router's lifecycle-gate verdict:

  • eventd/statedep.py:
  • DependencyIndex.from_entries(entries) — folds an eventd log into the
    dependency state.
  • DependencyIndex.verify_cold_start(consumer, entries) -> StaleDependency | None
    — None ⇒ the target may RUN (deliver); a StaleDependency record ⇒
    PAUSED(stale_dependency) (queue/refuse). This is the exact verdict the
    router's lifecycle gate must agree with.
  • StaleDependency.topology_epoch and each registration's topology_epoch —
    the epoch-currency input. The router reads the current epoch (read-only) and
    forwards it to target resolution; the refusal on a stale epoch is minted by
    S4/the supervisor as a ControlAck, not by the router (see §1.1, §3.4).
  • tests/spec/test_eventd.py — _mk_statedep_log(path, *, rewind=, evict=),
    _test_statedep(...) build canonical statedep eventd logs (fresh anchor,
    rewind, eviction, recovery). These fixtures are the seed corpus for the
    differential test (§5.2).

The router does not call gc_floor / compute_gc_floor; that is S5's verdict
(see §1.1). The oracle relationship is: given the same eventd log + the same
inbound frame, the Rust router's gate decision (deliver / queue) equals the
decision implied by verify_cold_start.

2.3 Upstream code dependency (WP3-S1, exists today)

  • protocol/hcp/hcpbin/src/lib.rs — hcpbin codec. Public API the router relies
    on (transitively, via S2's frame decoder): hcpbin::{Reader, Writer, Hash, DecodeError}, encode_uvarint/decode_uvarint, etc. The router operates on
    decoded frames and only re-enters hcpbin to mint the body of an outbound
    error frame.

3. Design

3.1 Crate layout

The router is the second module of the litany crate (the first, the frame layer,
is S2). Files to create:

protocol/hcp/litany/
  Cargo.toml                      # new crate: name="litany", deps: hcpbin (path)
  src/
    lib.rs                        # re-exports; (S2) creates this, S3 adds `pub mod router;`
    frame.rs                      # (S2) FrameHeader, FrameKind, decode/encode  — DEP, not created here
    router.rs                     # S3: Router, RouteVerdict, dispatch entry point
    correlation.rs                # S3: CorrelationTable (corr -> outstanding exchange)
    stream.rs                     # S3: StreamReassembler (stream/seq/end state machine)
    chapter.rs                    # S3: ChapterTable (chapter id -> open/closed, epoch)
    state_gate.rs                 # S3: EpochSource, TargetState (RFC 0004 lifecycle gate + epoch read)
    error.rs                      # S3: RouteError -> error_kind mapping (RFC 0003 §9.2.1)
  tests/
    route_unit.rs                 # S3: pure routing unit tests (no Python)
    route_differential.rs         # S3: differential vs eventd/statedep.py (§5.2)
  fixtures/
    statedep/                     # S3: eventd logs exported from tests/spec fixtures

frame.rs is an S2 deliverable; S3 lists it as a build dependency, does not author
it. If S2 has not landed FrameHeader, S3 defines a thin local shim against the
RFC 0003 §4.4.1 field set and a // TODO(S2): replace with litany::frame::FrameHeader.

3.2 Core types

// frame.rs (S2 — shown for reference only)
pub enum FrameKind { Request = 0, Response = 1, Event = 2, Control = 3, Error = 4 }
pub struct FrameHeader {
    pub kind:   FrameKind,
    pub corr:   [u8; 16],       // RFC 0003 §4.4.1: 16 raw bytes
    pub stream: Option<u64>,
    pub seq:    Option<u64>,
    pub end:    bool,
}
pub struct Frame<'a> { pub header: FrameHeader, pub body: &'a [u8] } // body opaque to router

// router.rs (S3)
pub enum RouteVerdict<'a> {
    Deliver  { target: TargetId, frame: Frame<'a> },
    Reply    { corr: [u8; 16],  frame: Frame<'a> },
    Queue    { target: TargetId, frame: Frame<'a> },   // RFC 0003 §2.1 / RFC 0005 §2.1
    Reassemble { stream: u64 },                          // chunk buffered, not terminal
    Refuse   { corr: [u8; 16], error_kind: ErrorKind },  // RFC 0003 §9.2.1
}

pub struct Router<E: EpochSource, S: TargetState, R: TargetResolver> {
    chapters:     ChapterTable,
    corr_table:   CorrelationTable,
    reassembler:  StreamReassembler,
    epoch:        E,
    target_state: S,
    resolver:     R,
}

impl<E, S, R> Router<E, S, R> { /* E:EpochSource, S:TargetState, R:TargetResolver */
    /// The single routing entry point. `chapter` is the chapter the connection
    /// established at `open` (RFC 0003 §6.2). Pure: no IO, no blocking.
    pub fn route<'a>(&mut self, chapter: ChapterId, frame: Frame<'a>) -> RouteVerdict<'a>;
}

3.3 The trait seams (this is what makes S3 codeable before S4 and S5)

S3 ships traits + test doubles; S5 / the supervisor provide real impls later.
This is the load-bearing design decision — without it, S3 hard-depends on
unbuilt sprints.

// state_gate.rs

/// Current topology epoch for a chapter (RFC 0004 §7.1: supervisor-assigned,
/// immutable per chapter). S3 ships a fixed/test double; S5 wires it to
/// agent-supervisord's epoch counter (via oraclefd / eventd).
pub trait EpochSource {
    fn current_epoch(&self, chapter: ChapterId) -> u64;
}

/// Lifecycle state of a routing target (RFC 0004 §6). S3 ships a HashMap double
/// seeded from `verify_cold_start` verdicts; S5 wires it to the supervisor's
/// boot state machine (STOPPED→…→RUNNING | PAUSED(stale_dependency)).
pub enum Lifecycle { Running, Paused(PauseReason) }
pub enum PauseReason { Operator, StaleDependency, GcDrain }
pub trait TargetState {
    fn lifecycle(&self, target: TargetId) -> Lifecycle;
}

/// (name, epoch) -> AgentId/TargetId (RFC 0005 §2.4; oraclefd surface, OPEN
/// in RFC 0005 §1). S3 ships a static roster double; S4/oraclefd provide real
/// resolution. For a Request to an unresolvable agent target the router refuses
/// with a §9.2.1 wire kind (`unknown_service`); RFC 0005's `unknown_target`
/// ControlRefusal is S4's ControlAck path, not a router-minted wire error.
pub trait TargetResolver {
    fn resolve(&self, name: &str, epoch: u64) -> Option<TargetId>;
}

3.4 The routing algorithm

Router::route(chapter, frame) is a pure function over injected state. Step order
is normative (early steps gate later ones):

route(chapter, frame):

  # --- A. Chapter binding (RFC 0003 §6.1–6.2) ---
  1. if chapter not open in chapters:
        return Refuse(frame.corr, chapter_closed)

  # --- B. Class dispatch (RFC 0002 §4.2/§4.3, kind 0–4) ---
  2. match frame.header.kind:
       Response | Error -> goto C (correlation; terminal Reply)
       Event            -> goto C (subscription corr); if multi-frame, D (stream);
                           terminal -> Reply to subscription handler (never E)
       Request          -> goto D (stream) then E (target gates) then deliver
       Control          -> goto F (control fast path)   # RFC 0003 §7.2

  # --- C. Correlation match (RFC 0003 §6.3) ---
  # Reached by Response, Error, Event. Request/Control skip C (no outstanding
  # local exchange to match) and go straight to D/E or F.
  3. lookup = corr_table.get(frame.corr)
     if none:
        return Refuse(frame.corr, invalid_correlation)   # benign-late; §9.2.1
     match kind:
        Response | Error -> corr_table.retire(frame.corr)  # exchange terminates
                            return Reply(frame.corr, frame)
        Event            -> # subscription continues; events may be multi-frame.
                            # Single-shot event: deliver now. Multi-frame: reassemble.
                            if frame.header.stream is None:
                               return Reply(frame.corr, frame)   # to subscription handler
                            (continue to D for stream reassembly; the TERMINAL
                             event chunk returns Reply(frame.corr, ...) — NOT step E:
                             events route by subscription corr, never by target name)

  # --- D. Stream reassembly (RFC 0003 §6.3, §4.4.1 decoder invariant) ---
  4. if frame.header.stream is Some(s):
        # parity: even = initiator-allocated, odd = responder (RFC 0003 §6.3)
        st = reassembler.get_or_open(s, frame.header.seq)
        if st == Retired:                     # id reused after end=true
            return Refuse(frame.corr, stream_reset)
        if frame.header.seq != st.expected_next_seq():
            reassembler.fault(s)
            return Refuse(frame.corr, seq_gap)   # in-order transport; gap = loss/bug
        st.accept(frame)
        if not frame.header.end:
            return Reassemble { stream: s }   # not terminal yet
        reassembler.retire(s)                 # end seen: id MUST NOT be reused
        # TERMINAL chunk:
        if kind == Event:
            return Reply(frame.corr, frame)   # to subscription handler; NOT step E
        # else (Request): fall through to E (target gates / delivery)
     # single-shot Request (no stream) also falls through to E.

  # --- E. RFC 0004 state-dependency gates (Request / terminal Request stream only) ---
  5. target = resolver.resolve(frame.target_name(), epoch.current_epoch(chapter))
     if none:
        # An unresolvable *agent* target for a Request is a wire error_kind.
        # NOTE: do NOT borrow RFC 0005's `unknown_target` ControlRefusal here —
        # that is a ControlAck reason, not a §9.2.1 wire error_kind. Map to the
        # closest §9.2.1 kind (`unknown_service`); see §7 open item on whether a
        # dedicated `unknown_target` wire error_kind should be added to hcp.wire.
        return Refuse(frame.corr, unknown_service)        # RFC 0003 §9.2.1

  6. # Lifecycle gate (RFC 0004 §6). Normal work to a stale target is queued.
     # This is the ONE genuinely-routing RFC 0004 gate (see §1.1 / §3.4 notes).
     # The epoch fence for state-dependency / control frames is NOT minted here:
     # epoch refusal produces a ControlAck{stale_epoch} (RFC 0005 §2.4/§7), which
     # is S4/supervisor's job — the router delivers control frames to S4 (step 7).
     match target_state.lifecycle(target):
        Running                       -> return Deliver { target, frame }
        Paused(Operator | GcDrain)    -> return Queue   { target, frame }  # RFC 0003 §2.1
        Paused(StaleDependency)       -> return Queue   { target, frame }  # RFC 0004 §6

  # --- F. Control fast path (RFC 0003 §7.2) ---
  7. # Control-class frames are NEVER queued behind data backpressure and bypass
     # the lifecycle queue gate (a paused agent must still answer ping/pause).
     # The router does NOT mint stale_epoch / unknown_target here: those are
     # RFC 0005 ControlRefusal reasons returned in a ControlAck (a response-class
     # frame), which S4/the supervisor produces after it validates the epoch
     # (RFC 0005 §2.4/§7) and resolves the target (RFC 0005 §2.4). The router's
     # job is to DELIVER the control frame to the control plane unblocked.
     return Deliver { target, frame }     # to the control plane (S4 dispatches verbs)

Notes the implementation must honor:

  • Step 6 vs step 7 is the §1.1 boundary made executable. The router queues
    normal work to a stale_dependency target but never blocks control frames
    — that is RFC 0003 §7.2 (liveness exempt from flow control) intersected with
    RFC 0004 §6 (no work to a stale target). The router does not decide when the
    pause clears; it only routes around it.
  • Epoch refusal is not a router-minted wire error. A control or
    state-dependency frame with a stale topology_epoch does NOT come back as an
    RFC 0003 wire error{stale_epoch} (no such §9.2.1 kind exists). It comes back
    as an RFC 0005 ControlAck{applied=false, reason=stale_epoch} (or, for a
    dependency frame, re-verification against the old epoch's graph — RFC 0004
    §7.1). Both are S4/supervisor outputs above the router. The router only
    delivers such frames; it never asserts or validates the epoch (epochs are
    supervisor-assigned — RFC 0004 §7).
  • Events route by subscription corr, never by target name. An event frame
    carries its subscription's corr (RFC 0002 §4.2); it is matched in step C and
    delivered (Reply) to the subscription handler. It never enters step E (target
    resolution / lifecycle gate) — that path is for Request work only.
  • Queue vs Refuse for a paused target is a policy knob: graceful pause
    (RFC 0003 §2.1, RFC 0005 §2.1) buffers up to a bound, then refuses. The
    overflow refusal is a resource-exhaustion / budget condition (preceptord/S4,
    RFC 0005 §Security, #28) — it is NOT an RFC 0003 §9.2.1 routing error_kind;
    S3 ships the bound as a constant with a // TODO to source the bound and the
    overflow disposition from S4 policy.
  • No reordering. seq is for gap detection only (RFC 0003 §6.3): the
    transport is in-order per direction (RFC 0003 T2), so a gap is loss/bug ⇒
    seq_gap, never a reorder buffer.
  • end retires the stream id (RFC 0003 §6.3); a later frame on a retired id is
    stream_reset.

3.5 Minting error frames

Every RouteVerdict::Refuse the router emits carries a genuine RFC 0003 §9.2.1
wire error_kind — chapter_closed, invalid_correlation, seq_gap,
stream_reset, or unknown_service (the closest §9.2.1 kind for an unresolvable
agent target; see §7 open item). The caller (the connection driver, S2) turns the
verdict into an error Votive Frame echoing corr with that error_kind
(RFC 0003 §9.2); minting the bytes uses hcpbin::Writer against the hcp.wire
error schema (RFC 0003 Appendix A). S3 provides
error::mint(corr, error_kind) -> Vec<u8> as a thin helper; the connection driver
owns send.

stale_epoch / unknown_target are deliberately absent from the router's
refusal set. They are RFC 0005 ControlRefusal reasons carried in a ControlAck
(a response-class frame), produced by S4/the supervisor — not RFC 0003 wire
error_kinds. The router delivers epoch/target-bearing control frames to S4 and
never mints these (see §3.4 step 7 and the epoch-refusal note).

4. Worked routing trace (matches RFC 0004 §5 checkpoint/rewind example)

Mirrors the RFC 0004 §5 worked example (producer P rewound; consumer C anchored
at step 95 > rewind point 50). The router sees C's next inbound work frame:

Setup: chapter open, current_epoch = N.
       C's lifecycle (from supervisor, seeded by verify_cold_start) = RUNNING
       while P's history is intact.

t0  Request{corr=X, target="C"} arrives:
      A:   chapter open                                 ok
      B:   kind=Request                                 -> D,E
      D:   no stream (single-shot)                       skip
      E.5: resolve("C", N) = TargetId(C)                ok
      E.6: lifecycle(C) = Running                       -> Deliver{C, ...}

t1  Operator rewinds P to step 50 (RFC 0005 RewindControl -> RFC 0004 §3.3).
      Supervisor pauses C, runs verify_cold_start(C, eventd_log):
        anchor step 95 > rewind 50  -> StaleDependency  -> lifecycle(C)=Paused(StaleDependency)

t2  Request{corr=Y, target="C"} arrives:
      E.6: lifecycle(C) = Paused(StaleDependency)       -> Queue{C, ...}
           (RFC 0004 §6: no work into corrupt history)

t3  Control{ping} to C arrives:
      F.7: control fast path, bypasses queue gate       -> Deliver{C, ping}
           (RFC 0003 §7.2: a paused agent still answers liveness)

t4  Operator re-anchors C at step 50; supervisor verify_cold_start(C)=None
      -> lifecycle(C)=Running. Queued frame (corr=Y) is re-routed -> Deliver.

The deliver/queue verdicts at t0, t2, t4 are exactly what verify_cold_start
returns (None→deliver, StaleDependency→queue); t3 is the RFC 0003 §7.2
override. This is the differential assertion (§5.2).

5. Test plan

5.1 Runs fully on M1 (cargo test, python3)

The router is pure async-free Rust over already-decoded frames: no namespace
syscalls, no transport, transport-agnostic. The kickoff's blanket "No local builds
on M1" applies to the transport/perf targets, not to this layer. Concretely:

Test How Where it runs
Routing unit tests (tests/route_unit.rs) cargo test -p litany M1 ✓
— class dispatch (5 kinds), corr match, invalid_correlation pure M1 ✓
— Event routes by subscription corr → Reply (single-shot and terminal multi-frame), never into target resolution (step E) pure M1 ✓
— stream reassembly: in-order accept, seq_gap, end retire, stream_reset, parity pure M1 ✓
— chapter binding: chapter_closed pure M1 ✓
— unresolvable Request target → unknown_service pure (double TargetResolver) M1 ✓

— lifecycle gate: Running→Deliver, Paused(StaleDependency)→Queue pure (double TargetState) M1 ✓

The Zig port is out of scope (RFC 0004 §8 order 8; WP4). No zig build is needed
for S3; if a Zig stub lands, zig build -Dtarget=x86_64-linux is compile-only on M1.

5.2 Differential test design (the verification spine)

Goal: prove the Rust lifecycle gate verdict agrees with eventd/statedep.py
(verify_cold_start), and that the epoch read from EpochSource matches the
oracle's epoch-currency check feeding target resolution.

  1. Export fixtures. A small Python helper (in tests/spec/, reusing
    _mk_statedep_log from tests/spec/test_eventd.py) writes the canonical
    statedep eventd logs — fresh anchor, post-rewind, post-eviction, recovered —
    as JSONL into protocol/hcp/litany/fixtures/statedep/*.jsonl. Committed.
  2. Oracle verdicts. For each fixture log L and each consumer C, compute the
    oracle: verify_cold_start(C, L) is None (deliver) vs StaleDependency
    (queue), plus epoch currency (reg.topology_epoch == current). Write these as
    a JSON sidecar *.expected.json.
  3. Rust side. route_differential.rs loads each fixture into the TargetState
    / EpochSource doubles (lifecycle = oracle verdict for that log), feeds a
    synthetic Request{target=C} frame, and asserts the RouteVerdict matches the
    sidecar: verify_cold_start returns None → Deliver; returns
    StaleDependency → Queue. (A stale-epoch control frame is delivered to S4,
    which mints the ControlAck{stale_epoch} — the router does not refuse it, so it
    is not part of this router-verdict assertion; it is covered by an S4 test.)
  4. Regression guard: add the RFC 0004 §5 trace (this spec §4) as a fixture —
    it must reproduce deliver(t0)/queue(t2)/deliver-control(t3)/deliver(t4).

7. Risks / open questions

  • FrameHeader ownership. If S2 slips, S3 needs the shim (§3.1). Mitigation:
    S3 defines FrameHeader against RFC 0003 §4.4.1 directly; merge with S2's on landing.
  • Where does target_name come from for a Request frame, and is there a wire
    unknown_target error_kind?     RFC 0005 §2.4 ties name resolution to the
    supervisor roster (oraclefd surface — OPEN in RFC 0005 §1 and §"Open
    questions"). RFC 0005's unknown_target is a ControlRefusal/ControlAck
    reason, not an RFC 0003 §9.2.1 wire error_kind. For a Request to an
    unresolvable agent target the router currently maps to unknown_service
    (§3.4 step 5). Open: should hcp.wire add a dedicated unknown_target wire
    error_kind for agent-level routing? Decide with S4/oraclefd. S3 abstracts
    resolution behind TargetResolver; the real source is S4/oraclefd.
  • Epoch on Request frames + who mints stale_epoch. RFC 0004/0005 attach
    topology_epoch to dependency and control frames, not to every Request. The
    router does not mint stale_epoch: a stale-epoch control or dependency frame is
    delivered to S4/the supervisor, which returns ControlAck{stale_epoch} or
    re-verifies against the old epoch (RFC 0004 §7.1). Plain Request frames carry no
    epoch and are gated only by the lifecycle gate (step 6). Confirm with S4 whether
    work frames should carry epoch at all (cross-node, RFC 0004 §7.1 eventual
    consistency) — currently they do not.
  • Queue bound + overflow class. Default bound is a placeholder constant;
    the refuse-on-overflow error_kind (resource-exhaustion) is a preceptord/S4
    budget concern (RFC 0005 §Security; #28). S3 ships a constant + TODO.
  • Cross-node routing (frame to a target on another host, Litany Wire between
    hosts) is RFC 0006 / the #16 mesh boundary — out of scope; single-connection
    routing only (RFC 0003 §6.5 v1: no cross-connection chapter resumption).

8. Acceptance criteria

S3 is done when ALL hold (all verifiable on M1):

  1. protocol/hcp/litany/ crate builds: cargo build -p litany (no transport
    features) is clean on M1.
  2. Router::route implements the §3.4 algorithm: every FrameKind (0–4) is
    dispatched; chapter binding, correlation, stream reassembly, the Event
    subscription-corr path, and the RFC 0004 lifecycle gate (plus the read-only
    epoch forward to resolution) are all reachable and unit-tested (§5.1 table —
    every row green).
  3. The three trait seams (EpochSource, TargetState, TargetResolver, §3.3)
    exist with documented signatures and test doubles; no RFC 0004 §4/§5/§6
    machinery (gc_floor, cold-start execution, DAG) is reimplemented in the crate
    (scope boundary §1.1 holds — verified by grep: no gc_floor symbol in src/).
  4. Frame-level error mapping covers exactly the RFC 0003 §9.2.1 wire error_kinds
    reachable from routing: chapter_closed, invalid_correlation, seq_gap,
    stream_reset, unknown_service — each has a unit test asserting the verdict.
    The router does NOT mint stale_epoch / unknown_target (those are RFC 0005
    ControlRefusal/ControlAck outputs owned by S4); a test asserts a stale-epoch
    control frame is delivered to the control plane, not refused.
  5. Control-class frames bypass the lifecycle queue gate (RFC 0003 §7.2) — explicit
    test (the §4 t3 case).
  6. Differential test (§5.2) passes: for every committed statedep fixture, the
    Rust router's deliver/queue verdict equals the verify_cold_start verdict from
    eventd/statedep.py. The RFC 0004 §5 worked trace (§4) is a committed
    regression fixture and reproduces t0/t2/t3/t4.
  7. The Python oracle still passes locally: python3 tests/spec/run_all.py green
    (statedep tests unaffected by fixture export).
  8. No dependency on dev-cx53 for any of 1–7. The ≤10µs/frame perf number is
    explicitly deferred to WP3-S6 (§5.3) and is NOT an S3 gate.
