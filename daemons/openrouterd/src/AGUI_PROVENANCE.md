# AG-UI core — provenance & status

The AG-UI agent-core substrate for the OPENFANG fan-out service. Five Zig modules + tests:

| Module | Purpose | Reviewed | Honest syscall note |
|--------|---------|----------|---------------------|
| `arena.zig` | offset-handle file-backed shared-memory arena (no raw ptr crosses to JS) | yes | uses only openat/read/write/mmap/munmap/close — all in the openrouterd 22-syscall seccomp list; `unlink` cleanup gap documented (Option A/B/C) |
| `agui.zig` | AG-UI SSE event core (agent/chunk · tool/call · interaction/approval) + EventQueue | yes (adversarial) | pure core is syscall-free; a real `serve()` needs bind/listen/accept — **NOT** in the 22-list → broker runs unsandboxed |
| `shell.zig` | isolated subprocess executor (capture stdout/stderr, timeout) | yes (adversarial) | fork/execve/wait4/kill — **NOT** in the 22-list → must run as a separate unsandboxed broker |
| `memgraph.zig` | AF_UNIX client to codebase-memory-mcp | controller | socket/connect/sendto/recvfrom — in the 22-list (client fits) |
| `patch.zig` | atomic line-block patch editor | controller | openat/read/write/close in-list; true-atomic rename caveat documented |
