## Conductor: models choose their own

Every `createVakedAgent()` call now auto-routes prompts to the best model. No configuration needed.

```typescript
const agent = createVakedAgent();

// "Write a Zig 0.16 HTTP server" → Claude Opus (code keyword)
// "Explain capability graphs"     → DeepSeek V4 Pro (default)
// "Brainstorm daemon names"        → Gemini Flash (creative keyword)
```

The routing is keyword-based — zero latency, zero API cost:

| Keywords | Model | Cost/1M |
|----------|-------|---------|
| code, write, implement, fix, debug, test, refactor, optimize, review | Claude Opus 4.8 | $15/$75 |
| creative, brainstorm, design, story | Gemini 2.5 Flash | $0/$0.60 |
| everything else | DeepSeek V4 Pro | $0.27/$0.27 |

## Context7 pre-scan: 19 patterns, zero config

Every prompt is scanned for API/library mentions before reaching the model. When detected, up to 2K tokens of live Context7 documentation are injected:

```
User: "How do I use std.Build in Zig 0.16?"
        ↓
[ctx7:prescan] detected: zig → injected 1847 tokens (~7.4K)
        ↓
Enriched prompt sent to model:
  ## Context7: zig (live docs - authoritative)
  const std = @import("std");
  pub fn build(b: *std.Build) void { ... }
  ...
  ---
  User: "How do I use std.Build in Zig 0.16?"
```

19 library patterns auto-detected: zig, nixpkgs, tauri, react, cloudflare, nodejs, rust, ebpf, monaco, python, go, docker, kubernetes, typescript, git, sql, vite, langchain.

The model never sees stale documentation again.

## Atlas: the daemon

`openrouterd` is a Zig 0.16 native daemon that replaces the Node.js/Bun runtime for production deployments:

```bash
$ ./zig-out/bin/openrouterd
info: openrouterd :9090 model=deepseek/deepseek-v4-pro genesis=7c242080

$ curl localhost:9090/health
{"status":"ok","genesis":"7c242080","nickname":"Atlas","defaultModel":"deepseek/deepseek-v4-pro"}

$ curl -X POST localhost:9090/ -d '{"prompt":"What is Zig?","auto":true}'
{"genesis":"7c242080","model":"deepseek/deepseek-v4-pro","content":"Zig is a systems..."}
```

| Layer | Mechanism |
|-------|-----------|
| **I/O** | Raw linux sockets (io_uring planned) |
| **Memory** | Arena allocator, zero fragmentation |
| **Security** | seccomp BPF — 22 syscalls allowed |
| **Conductor** | Auto-routing built into the daemon |
| **Endpoints** | `/health`, `/models`, `POST /` (prompt) |
| **Streaming** | SSE (`text/event-stream`) |
| **Deploy** | systemd unit with full hardening |

Atlas carries the swarm's LLM load. Named after the Titan who holds up the sky.

## Langfuse: every call traced, nothing blocks

All LLM calls through `createVakedAgent()` are automatically traced to Langfuse. Credentials come from the GitHub CI Environment `ci`. The guard pattern ensures tracing never blocks:

```typescript
// In createVakedAgent(), every ask()/code()/review() call:
const startTime = Date.now();
const text = await result.getText();

// Guarded — no-op when LANGFUSE_SECRET_KEY is unset
traceCallModelResult({
  model: entry.id,
  input: prompt,
  output: text,
  promptTokens: response.usage?.inputTokens ?? 0,
  completionTokens: response.usage?.outputTokens ?? 0,
  latencyMs: Date.now() - startTime,
  agentName: "vaked-agent-ask",
});
```

Secrets from CI, guarded, advisory. Never blocks. Matching the Rust agents' convention.

## Multi-runtime: Bun, Deno, Node.js

```bash
# Bun (mimalloc built-in — recommended for production)
bun run dist/cli.js "prompt"

# Node.js (tuned V8 flags)
node --turbo-fast-api-calls --max-old-space-size=4096 dist/cli.js "prompt"

# Deno (jemalloc on Linux)
deno run --allow-env --allow-net --allow-read dist/cli.js "prompt"
```

| Runtime | Allocator | Best for |
|---------|-----------|----------|
| Bun | mimalloc (built-in) | Production agents, long loops |
| Node.js | system malloc + tuned heap | General use |
| Deno | jemalloc (Linux) | Sandboxed agents |

Complete tuning guide in `RUNTIME.md` and `MIMALLOC.md`.
